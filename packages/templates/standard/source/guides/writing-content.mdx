# Writing content

Clarify pages are MDX files. You can start with plain Markdown and add components when a page needs richer structure.

## Frontmatter

Use frontmatter to control page metadata.

```mdx title="page.mdx"
---
title: Writing content
description: Explain how authors write documentation.
keywords:
  - mdx
  - docs
---

# Writing content
```

## Callouts

<Callout title="Use callouts for important notes">
  Callouts are useful for setup requirements, warnings, migration notes, and product constraints.
</Callout>

<Note>
  `Note` is a lightweight MDX primitive for contextual information inside a section.

Use code groups when users need package-manager or framework variants.


```bash title="npm"
npm install @clarify-labs/cli
```

## Properties

Use properties for configuration, component props, environment variables, or response fields.

<Properties>
  <Property name="title" type="string">
    Site title used in the header and SEO metadata.
  </Property>
  <Property name="description" type="string">
    Default description used for pages without a custom description.
  </Property>
  <Property name="tabs" type="ClarifyTabsConfig">
    Top-level navigation areas. Each tab owns its sidebar groups and pages.
  </Property>
</Properties>
