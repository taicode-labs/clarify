# Build documentation with Clarify

This standard starter includes a practical site configuration, navigation, MDX pages, built-in components, and a changelog. Use it as a clean starting point for product docs, internal docs, or project guides.

<Callout title="Starter tip">
  All editable content lives in `{{contentDir}}`. Start with `clarify.ts` for site structure, then edit the MDX pages.
</Callout>

## What is included

<CardGroup cols={2}>

## Quick edits

1. Update `clarify.ts` with your product name, theme, navbar, footer, tabs, and sidebar pages.
2. Replace pages under `{{contentDir}}/guides` with your real documentation.
3. Update `{{contentDir}}/changelog.mdx` with your release notes.
4. Run `pnpm dev` to preview and `pnpm build` to create the static site.

      </Property>

    ```ts title="clarify.ts"
      title: 'Your Docs',
      tabs: [
        {
          tab: 'Docs',
          pages: 'FileTree',
        },
      ],
    })
    ```
