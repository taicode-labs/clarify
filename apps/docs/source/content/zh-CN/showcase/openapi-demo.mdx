# OpenAPI 文档渲染演示

本页展示 Clarify 渲染 OpenAPI 规范的效果。

---

## 自动生成的 API 页面

将 `api.openapi.json` 放入 `source/content/` 目录后，Clarify 会自动生成路由 `/api`，并使用 `OpenApiPage` 组件渲染完整的 API 文档。这个示例规范覆盖 OpenAPI 3.1 的常见复杂结构，包括 `$ref`、参数、请求体、响应、headers、examples、links、callbacks、webhooks、security schemes、多 content type、schema 组合和 discriminator。

```
标题：Clarify Complete OpenAPI Example
版本：1.0.0

GET     /pages              List pages
POST    /pages              Create a page
OPTIONS /pages              Describe page collection capabilities
GET     /pages/{pageId}     Get a page
PUT     /pages/{pageId}     Replace a page
PATCH   /pages/{pageId}     Patch a page
DELETE  /pages/{pageId}     Delete a page
HEAD    /pages/{pageId}     Check if a page exists
POST    /assets             Upload an asset
GET     /search             Search pages
POST    /exports            Create an export
GET     /system/health      Get service health
TRACE   /system/health      Trace health request
```

---

## 嵌入单个接口

在任意 MDX 页面中，使用 `OpenApiEndpoint` 组件嵌入特定接口。**无需导入，Clarify 的内置组件在 MDX 中可直接使用。**

```mdx
# 页面管理 API

<OpenApiEndpoint specPath="/api" path="/pages" method="get" />
```

渲染效果：

<OpenApiEndpoint specPath="/api" path="/pages" method="get" />

---

## 多媒体类型和多示例接口

`POST /exports` 用于覆盖更复杂的 OpenAPI 示例组合：请求体有 JSON 与 YAML 两种 media type，每种 media type 下都有多个 examples；响应有 `202` 与 `400` 两种 status code，且成功响应同时提供 JSON 与纯文本 media type。

```mdx
<OpenApiEndpoint specPath="/api" path="/exports" method="post" />
```

渲染效果：

<OpenApiEndpoint specPath="/api" path="/exports" method="post" />
