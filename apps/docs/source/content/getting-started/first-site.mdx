# 创建你的第一个站点

在上一节完成安装后，本节将带你深入了解 Clarify 的目录结构和约定，创建一个更完整的文档站点。

---

## 目录结构

一个典型的 Clarify 项目如下：

```
my-docs/
├── clarify.json          # 站点配置（标题、描述、主题等）
├── package.json          # dev/build 脚本
└── source/
    └── content/          # 文档内容目录
        ├── index.mdx     # 首页
        ├── guide/
        │   ├── index.mdx
        │   └── advanced.mdx
        └── api.mdx
```

---

## 编写页面

Clarify 支持 **Markdown** 和 **MDX**。你可以在 MDX 中直接使用 JSX：

```mdx
# 欢迎使用

这是普通 Markdown 段落。

<div className="bg-blue-100 p-4 rounded">
  这是一个 React 组件！
</div>
```

### Frontmatter

每个内容文件顶部可以添加 YAML frontmatter，用于设置页面元数据：

```mdx
---
title: 页面标题
description: 页面描述，用于 SEO
---

# 正文内容
```

---

## 自动路由

Clarify 按照 `source/content` 下的文件结构自动生成路由：

| 文件路径 | 路由路径 |
|----------|----------|
| `index.mdx` | `/` |
| `guide.mdx` | `/guide` |
| `guide/index.mdx` | `/guide` |
| `guide/advanced.mdx` | `/guide/advanced` |

你不需要手动配置路由表，新增文件即新增页面。

---

## 本地开发

```bash
pnpm dev
```

---

## 构建生产版本

```bash
pnpm build
```

构建产物输出到 `output/` 目录（可通过 `clarify build --output <dir>` 修改），每个路由都有独立的 `index.html`，可直接部署到任何静态托管服务。
