# 配置系统

Clarify 采用双层配置设计，将**内容展示**和**构建行为**明确分离。

---

## 配置分层

| 层级 | 受众 | 文件 | 职责 |
|------|------|------|------|
| **项目配置** | 内容作者、技术写作者 | `clarify.json` | 站点展示什么 |
| **插件配置** | 前端开发者、构建工程师 | `vite.config.ts` | 站点如何构建和运行 |

> 内容创作者不应修改 `vite.config.ts`，开发者不应频繁修改 `clarify.json`。

---

## clarify.json

放置在项目根目录（与 `vite.config.ts` 同级），是面向内容作者的配置文件。

### 支持的字段

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `title` | `string` | 站点标题 | `"Clarify Docs"` |
| `description` | `string` | 站点描述，用于 SEO | `""` |
| `logo` | `string` | 品牌 Logo 路径 | — |
| `routeBase` | `string` | 站点基础路径 | `"/"` |
| `theme` | `object` | 主题配置 | `{}` |

### 示例

```json
{
  "title": "Clarify 文档",
  "description": "Clarify 官方文档 — 开源文档发布工具",
  "logo": "/logo.svg",
  "routeBase": "/docs/",
  "theme": {
    "primary": "#0ea5e9"
  }
}
```

## 插件选项

在 `vite.config.ts` 中通过 `clarifyPlugin(options)` 传入，控制构建时行为。

### 支持的字段

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `docsRoot` | `string` | 内容文件根目录 | `"source/content"` |
| `routeBase` | `string` | 部署路径，覆盖 `clarify.json` | `"/"` |
| `outPath` | `string` | 构建输出目录 | `"output"` |
| `include` | `FilterPattern` | MDX 处理包含规则 | `"**/*.mdx"` |
| `exclude` | `FilterPattern` | MDX 处理排除规则 | — |
| `plugins` | `ClarifyPlugin[]` | 扩展插件数组 | — |

### 示例

```typescript
clarifyPlugin({
  docsRoot: 'docs',
  routeBase: '/staging/docs/',
  outPath: 'build',
});
```

---

## 优先级

当同一字段在两层都配置时，优先级为：

```
vite.config.ts（插件选项）> clarify.json > 默认值
```

这允许开发者在不修改 `clarify.json` 的情况下覆盖作者设置，例如为 staging 环境使用不同的 `routeBase`。

## 从 Mintlify 迁移

Mintlify 的 `docs.json` 与 Clarify 的 `clarify.json` 字段基本兼容：

| Mintlify | Clarify | 说明 |
|----------|---------|------|
| `name` | `title` | 同义 |
| `description` | `description` | 直接对应 |
| `logo` | `logo` | 直接对应 |
| `colors.primary` | `theme.primary` | 路径调整 |

Clarify 会接受未知的 Mintlify 字段并静默忽略（附带警告），提供平滑的迁移路径。
