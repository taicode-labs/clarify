# 路线图

Clarify 采用分阶段迭代开发，每个阶段聚焦一组核心能力。

---

## Phase 1 — 最小可用产品（当前）

> ✅ 已完成

### 目标

构建一个可用的文档发布工具，支持 MDX 渲染和静态站点生成。

### 已实现功能

- [x] MDX 编译与渲染
- [x] 基于文件系统的约定式路由
- [x] 构建时静态生成（SSG）
- [x] 客户端 Hydration
- [x] Tailwind CSS 集成
- [x] 双层配置系统（`clarify.json` + 插件选项）
- [x] 基础主题定制（主色调）
- [x] 顶部导航配置
- [x] OpenAPI 集成 — 自动生成 API 文档页面或嵌入单个接口
- [x] 自定义导航 — 通过 `pages` 字段配置侧边栏结构

---

## Phase 2 — 增强体验（开发中）

### 目标

在 Phase 1 基础上增加高级功能，提升文档站点的交互性和可用性。

### 计划功能

- [ ] **全文搜索** — 基于内容的客户端搜索功能
- [ ] **国际化（i18n）** — 多语言文档支持
- [ ] **深色模式** — 自动/手动切换的深色主题
- [ ] **社交分享** — Open Graph 和 Twitter Card 支持
- [ ] **编辑链接** — "在 GitHub 上编辑此页" 功能
- [ ] **自定义 MDX 组件** — 注册项目专属的 React 组件到 MDX 作用域

---

## Phase 3 — 生态扩展（规划中）

### 目标

将 Clarify 扩展为更完整的文档生态系统。

### 设想功能

- [ ] **搜索后端** — 基于 Algolia / Pagefind 的增强搜索
- [ ] **版本管理** — 多版本文档并行展示
- [ ] **评论系统** — 基于 Giscus / Utterances 的页面评论
- [ ] **插件市场** — 第三方插件的发现和安装机制
- [ ] **CLI 工具** — 脚手架、内容校验、预览等命令行工具
- [ ] **VS Code 扩展** — 内容预览和自动补全
