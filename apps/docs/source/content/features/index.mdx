# 功能特性

Clarify 提供了一套完整的文档站点构建能力，从写作到部署，覆盖文档生命周期的每个环节。

---

## 功能一览

| 功能 | 说明 | 状态 |
|------|------|------|
| [MDX 写作](/features/mdx) | 在 Markdown 中使用 React 组件 | ✅ 已支持 |
| [文件路由](/features/routing) | 基于目录结构自动生成路由 | ✅ 已支持 |
| [静态生成](/features/ssg) | 构建时预渲染 HTML，SEO 友好 | ✅ 已支持 |
| [主题与样式](/features/theming) | Tailwind CSS + 自定义主题 | ✅ 已支持 |
| [OpenAPI 集成](/features/openapi) | 内嵌 API 参考文档 | ✅ 已支持 |

---

## 设计理念

Clarify 的设计遵循三个核心原则：

1. **作者优先** — 内容创作者只需关心 Markdown 和 `clarify.json`，不需要了解构建细节。
2. **约定优于配置** — 文件即路由，目录即结构，减少不必要的配置。
3. **渐进增强** — 基础功能开箱即用，高级功能按需启用。
