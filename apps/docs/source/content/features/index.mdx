# 功能概览

这里按能力概览 Clarify 已支持的功能。如果你正在完成具体任务，可以优先从左侧的“编写内容”“配置站点”“API 文档”“发布与迁移”章节进入。

## 设计理念

Clarify 的设计遵循三个核心原则：

1. **作者优先** — 内容创作者只需关心 Markdown 和 `clarify.json`，不需要了解构建细节。
2. **约定优于配置** — 文件即路由，目录即结构，减少不必要的配置。
3. **渐进增强** — 基础功能开箱即用，高级功能按需启用。
