# 配置导航与侧边栏

Clarify 的导航结构基于文件系统自动生成，同时支持通过配置进行微调。

---

## 顶部导航

在 `clarify.json` 中配置顶部导航链接：

```json
{
  "navbar": {
    "links": [
      {
        "label": "文档",
        "href": "/getting-started"
      },
      {
        "label": "GitHub",
        "href": "https://github.com/taicode-labs/clarify",
        "external": true
      }
    ]
  }
}
```

| 字段 | 说明 |
|------|------|
| `label` | 导航项显示文字 |
| `href` | 链接地址 |
| `external` | 是否在新标签页打开 |

---

## 侧边栏

侧边栏根据文件目录结构自动生成，遵循以下规则：

1. **目录层级** = 侧边栏分组
2. **`index.mdx`** = 分组的入口页面
3. **同级文件** = 按字母顺序排列

### 排序

在 `pages` 手动配置模式下，页面按照数组中定义的顺序排列。

在 `FileTree` 自动模式下，页面按文件名字母顺序排列。

### 隐藏页面

目前 Clarify 不提供隐藏页面的 frontmatter 选项。如需隐藏，请使用 `pages` 手动配置模式，不将对应页面加入 `items` 数组即可。页面文件仍然存在，可通过 URL 直接访问。

---

## 自定义侧边栏（Phase 2）

未来版本将支持完全自定义侧边栏结构：

```json
{
  "sidebar": [
    {
      "group": "快速开始",
      "pages": [
        "/getting-started/installation",
        "/getting-started/first-site"
      ]
    },
    {
      "group": "功能特性",
      "pages": [
        "/features/mdx",
        "/features/routing"
      ]
    }
  ]
}
```

> 此功能正在开发中，请关注 [路线图](/roadmap)。
