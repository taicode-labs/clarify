# Clarify 配置参考

推荐使用 `clarify.ts` 作为项目配置文件。`clarify.js` 和 `clarify.json` 也受支持，加载优先级为：

```
clarify.ts > clarify.js > clarify.json
```

`clarify.json` 适合不需要插件函数的简单项目或旧项目兼容；新项目建议使用 `clarify.ts`。

---

## 配置分层

Clarify 将**项目配置**和**运行参数**明确分离：

| 层级 | 受众 | 入口 | 职责 |
|------|------|------|------|
| **项目配置** | 内容作者、技术写作者、扩展开发者 | `clarify.ts/js/json` | 站点展示和构建期扩展 |
| **CLI 参数** | 开发者、构建环境 | `clarify dev/build` | 站点如何运行和输出 |

> 内容创作者主要维护 `clarify.ts` 和 `source/content`；开发者通过 CLI 参数控制端口、内容目录和输出目录。

## 完整字段列表

### 字段列表

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `title` | `string` | 站点标题 | `"Clarify Docs"` |
| `description` | `string` | 站点描述，用于 SEO | `""` |
| `logo` | `string` | 品牌 Logo 路径 | — |
| `favicon` | `string` | 站点 favicon 路径 | — |
| `routePrefix` | `string` | 站点基础路径 | `"/"` |
| `theme` | `object` | 主题配置 | `{}` |
| `navbar` | `array` | 顶部导航栏链接 | — |
| `banner` | `object` | 顶部横幅通知 | — |
| `footer` | `object` | 页脚配置 | — |
| `pages` | `array \| string` | 侧边栏导航结构 | `"FileTree"` |

### `theme` 对象

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `primary` | `string` | 主题主色（CSS 颜色值） | `"#0ea5e9"` |

## 示例配置

```ts
import { defineConfig } from '@clarify/cli'

export default defineConfig({
  title: 'Clarify 文档',
  description: 'Clarify 官方文档 — 开源文档发布工具',
  logo: '/logo.svg',
  favicon: '/favicon.ico',
  routePrefix: '/docs/',
  theme: {
    primary: '#0ea5e9',
  },
  navbar: {
    links: [
      { label: '指南', href: '/docs/getting-started/' },
      { label: 'GitHub', href: 'https://github.com/clarify/docs', external: true },
    ],
  },
  footer: {
    copyright: '© 2024 Clarify Team',
  },
  plugins: [
    // searchPlugin(),
  ],
})
```

## `pages` 侧边栏导航

`pages` 控制侧边栏的导航结构。支持两种配置方式：

### 1. 自动生成（默认）

```ts
export default defineConfig({
  pages: 'FileTree',
})
```

Clarify 会根据 `source/content` 目录的文件结构自动生成侧边栏，包含所有 `.mdx` 和 `.openapi.json` 文件。

### 2. 手动配置

```ts
export default defineConfig({
  pages: [
    {
      group: '开始使用',
      pages: [
        { page: 'getting-started/installation' },
        { page: 'getting-started/quick-start' },
      ],
    },
    {
      group: 'API 文档',
      pages: [
        { openapi: 'api', title: 'REST API' },
      ],
    },
  ],
})
```

- `page` — 引用一个 MDX 文件，不需要 `.mdx` 后缀
- `openapi` — 引用一个 OpenAPI 规范文件，不需要 `.openapi.json` 后缀，可指定 `title` 覆盖默认标题

## 优先级

Clarify 会先按 `clarify.ts > clarify.js > clarify.json` 查找配置文件，然后再应用 CLI 参数覆盖运行时选项：

```
CLI 参数 > clarify.ts/js/json > 默认值
```

这允许开发者在不修改配置文件的情况下覆盖运行时设置，例如为 CI 构建使用不同的输出目录。
