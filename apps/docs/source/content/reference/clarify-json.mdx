# clarify.json 配置参考

`clarify.json` 是面向内容作者的配置文件，位于项目根目录，控制站点的展示内容。

---

## 配置分层

Clarify 将**内容展示**和**运行/构建参数**明确分离：

| 层级 | 受众 | 入口 | 职责 |
|------|------|------|------|
| **项目配置** | 内容作者、技术写作者 | `clarify.json` | 站点展示什么 |
| **CLI 参数** | 开发者、构建环境 | `clarify dev/build` | 站点如何运行和输出 |

> 内容创作者主要维护 `clarify.json` 和 `source/content`，开发者通过 CLI 参数控制端口、内容目录和输出目录。

---

## 完整字段列表

### 字段列表

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `title` | `string` | 站点标题 | `"Clarify Docs"` |
| `description` | `string` | 站点描述，用于 SEO | `""` |
| `logo` | `string` | 品牌 Logo 路径 | — |
| `favicon` | `string` | 站点 favicon 路径 | — |
| `routePrefix` | `string` | 站点基础路径 | `"/"` |
| `theme` | `object` | 主题配置 | `{}` |
| `navbar` | `array` | 顶部导航栏链接 | — |
| `banner` | `object` | 顶部横幅通知 | — |
| `footer` | `object` | 页脚配置 | — |
| `pages` | `array \| string` | 侧边栏导航结构 | `"FileTree"` |

### `theme` 对象

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `primary` | `string` | 主题主色（CSS 颜色值） | `"#0ea5e9"` |

---

## 示例配置

```json
{
  "title": "Clarify 文档",
  "description": "Clarify 官方文档 — 开源文档发布工具",
  "logo": "/logo.svg",
  "favicon": "/favicon.ico",
  "routePrefix": "/docs/",
  "theme": {
    "primary": "#0ea5e9"
  },
  "navbar": [
    { "label": "指南", "href": "/docs/getting-started/" },
    { "label": "GitHub", "href": "https://github.com/clarify/docs", "external": true }
  ],
  "footer": {
    "copyright": "© 2024 Clarify Team"
  }
}
```

## `pages` 侧边栏导航

`pages` 控制侧边栏的导航结构。支持两种配置方式：

### 1. 自动生成（默认）

```json
{
  "pages": "FileTree"
}
```

Clarify 会根据 `source/content` 目录的文件结构自动生成侧边栏，包含所有 `.mdx` 和 `.openapi.json` 文件。

### 2. 手动配置

```json
{
  "pages": [
    {
      "label": "开始使用",
      "items": [
        { "page": "getting-started/installation.mdx" },
        { "page": "getting-started/quick-start.mdx" }
      ]
    },
    {
      "label": "API 文档",
      "items": [
        { "openapi": "api.openapi.json", "title": "REST API" }
      ]
    }
  ]
}
```

- `page` — 引用一个 MDX 文件
- `openapi` — 引用一个 OpenAPI 规范文件，可指定 `title` 覆盖默认标题

## 优先级

当同一项可由 `clarify.json` 和 CLI 参数共同影响时：

```
CLI 参数 > clarify.json > 默认值
```

这允许开发者在不修改 `clarify.json` 的情况下覆盖运行时设置，例如为 CI 构建使用不同的输出目录。
