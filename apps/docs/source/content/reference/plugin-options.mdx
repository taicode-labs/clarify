# Vite 插件选项参考

`@clarify/vite-plugin` 的 `clarifyPlugin()` 函数接收一个选项对象，用于控制构建时行为。

---

## 选项列表

### 选项列表

| 选项 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `routePrefix` | `string` | 部署路径，覆盖 `clarify.json` 中的设置 | `"/"` |
| `rootDirectory` | `string` | 内容文件根目录 | `"source/content"` |
| `outputDirectory` | `string` | 构建输出目录，覆盖 Vite 的 `build.outDir` | `"output"` |
| `include` | `string \| RegExp \| Array` | MDX 处理包含规则 | `"**/*.mdx"` |
| `exclude` | `string \| RegExp \| Array` | MDX 处理排除规则 | — |
| `plugins` | `ClarifyPlugin[]` | 扩展插件数组（翻译、搜索等） | — |

---

## 示例

### 基本配置

```typescript
import { defineConfig } from 'vite';
import { clarifyPlugin } from '@clarify/vite-plugin';

export default defineConfig({
  plugins: [
    clarifyPlugin(),
  ],
});
```

### 自定义内容目录

```typescript
clarifyPlugin({
  rootDirectory: 'docs',
});
```

### Staging 环境配置

```typescript
clarifyPlugin({
  routePrefix: '/staging/docs/',
  outputDirectory: 'build',
});
```

### 排除特定文件

```typescript
clarifyPlugin({
  exclude: ['**/draft/**/*.mdx', '**/_*.mdx'],
});
```

---

## 优先级

当同一字段在插件选项和 `clarify.json` 中都配置时：

```
插件选项 > clarify.json > 默认值
```

例如，如果 `clarify.json` 中 `routePrefix` 为 `"/"`，但插件选项中设置为 `"/staging/"`，最终生效的是 `"/staging/"。

---

## 构建行为说明

`clarifyPlugin` 在构建时会：

1. 扫描 `rootDirectory` 目录，发现所有 `.mdx` 和 `.openapi.json/.openapi.yaml/.openapi.yml` 文件
2. 为每个文件生成对应的路由和导航结构
3. 在构建时通过 `renderToString` 生成静态 HTML
4. 客户端通过 `hydrateRoot` 接管为单页应用

你可以通过 `include`/`exclude` 过滤不需要处理的文件，通过 `plugins` 接入翻译、搜索等扩展功能。
