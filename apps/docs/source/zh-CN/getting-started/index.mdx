# 快速开始

这篇文档用一条最短路径说明 Clarify 如何工作：安装 CLI、创建内容、启动本地站点、构建静态产物。

Clarify = MDX 内容 + `clarify.ts` 配置 + OpenAPI 规范 + 静态站点生成。你主要维护 `source/` 和配置文件，Clarify 负责开发服务器、路由、渲染和构建。

<Steps>
  <Step title="准备环境">

    需要：

    - Node.js ≥ 20
    - pnpm ≥ 9

    使用官方初始化命令创建新项目：

    <CodeGroup>

    ```bash title="pnpm"
    pnpm dlx @clarify-labs/cli init my-docs
    cd my-docs
    ```

    ```bash title="npm"
    npx @clarify-labs/cli init my-docs
    cd my-docs
    ```

    ```bash title="yarn"
    yarn dlx @clarify-labs/cli init my-docs
    cd my-docs
    ```

    </CodeGroup>

    脚手架会自动帮你写入 `package.json` 脚本：

    ```json title="package.json"
    {
      "scripts": {
        "dev": "clarify dev",
        "build": "clarify build --output output"
      }
    }
    ```

  </Step>
  <Step title="添加站点配置">

    默认使用 `clarify.json`。需要类型提示或接入插件时，可以切换为 `clarify.ts`。

    <CodeGroup>

      ```json title="clarify.json"
      {
        "title": "我的文档",
        "description": "基于 Clarify 构建的文档站点",
        "navbar": {
          "links": [
            { "label": "文档", "href": "/getting-started" },
            { "label": "GitHub", "href": "https://github.com/example/project", "external": true }
          ]
        }
      }
      ```

      ```ts title="clarify.ts"
      import { defineConfig } from '@clarify-labs/cli'

      export default defineConfig({
        title: '我的文档',
        description: '基于 Clarify 构建的文档站点',
        navbar: {
          links: [
            { label: '文档', href: '/getting-started' },
            { label: 'GitHub', href: 'https://github.com/example/project', external: true },
          ],
        },
      })
      ```

    </CodeGroup>

    完整字段见 [配置参考](/reference/clarify-config)。

  </Step>
  <Step title="编写第一批页面">

    Clarify 默认读取 `source/` 目录。新增 `.mdx` 文件就是新增页面。

    <FileTree aria-label="新项目骨架">
      <FileTreeItem name="my-docs" type="folder">
        <FileTreeItem name="clarify.json" />
        <FileTreeItem name="package.json" />
        <FileTreeItem name="source" type="folder">
          <FileTreeItem name="index.mdx" />
          <FileTreeItem name="getting-started.mdx" />
        </FileTreeItem>
      </FileTreeItem>
    </FileTree>

    `source/index.mdx`：

    ```mdx title="source/index.mdx"
    ---
    title: 首页
    description: 我的产品文档首页
    ---

    # 欢迎使用

    这里是文档首页。
    ```

    `source/getting-started.mdx`：

    ```mdx title="source/getting-started.mdx"
    ---
    title: 快速开始
    ---

    # 快速开始

    这里是第一篇指南。
    ```

    路由会自动生成：

    | 文件 | 路由 |
    |------|------|
    | `source/index.mdx` | `/` |
    | `source/getting-started.mdx` | `/getting-started` |
    | `source/guides/index.mdx` | `/guides` |
    | `source/guides/auth.mdx` | `/guides/auth` |

  </Step>
  <Step title="启动开发服务器">

    优先通过项目脚本启动，这样无论 CLI 是全局安装还是项目本地依赖，使用方式都一致：

    <CodeGroup>

    ```bash title="pnpm"
    pnpm dev
    ```

    ```bash title="npm"
    npm run dev
    ```

    ```bash title="yarn"
    yarn dev
    ```

    </CodeGroup>

    打开终端输出的本地地址。你修改 MDX、配置或 OpenAPI 文件后，开发服务器会重新生成页面。

  </Step>
  <Step title="构建生产版本">

    <CodeGroup>

    ```bash title="pnpm"
    pnpm build
    ```

    ```bash title="npm"
    npm run build
    ```

    ```bash title="yarn"
    yarn build
    ```

    </CodeGroup>

    默认输出到 `output/`：

    <FileTree aria-label="构建输出目录">
      <FileTreeItem name="output" type="folder">
        <FileTreeItem name="index.html" />
        <FileTreeItem name="getting-started" type="folder">
          <FileTreeItem name="index.html" />
        </FileTreeItem>
        <FileTreeItem name="assets" type="folder" />
        <FileTreeItem name="llms.txt" />
      </FileTreeItem>
    </FileTree>

    这些都是静态文件，可以部署到 Vercel、Netlify、GitHub Pages、S3、Nginx 或任意静态托管服务。

  </Step>
</Steps>
