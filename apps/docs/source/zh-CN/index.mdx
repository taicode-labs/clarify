# Clarify

开源文档发布工具，为 MDX 和 OpenAPI 而生。

Clarify 让你用熟悉的 Markdown + React 组件编写文档，自动生成为 SEO 友好的静态站点。

## 核心特性

| 特性 | 说明 |
|------|------|
| **MDX 支持** | 在 Markdown 中直接使用 React 组件，支持代码高亮、自定义组件 |
| **文件路由** | 按照文件目录结构自动生成路由，无需手动配置 |
| **静态生成 (SSG)** | 构建时为每个页面生成独立的 HTML，首屏加载快且对搜索引擎友好 |
| **内置样式系统** | 由 Clarify 渲染器提供一致的文档 UI，并支持基础品牌色配置 |
| **OpenAPI 集成** | 从同一份规范生成 API Reference，并在 MDX 指南中嵌入单个接口 |

## 快速预览

一个典型的 Clarify 文档站点只需要安装 CLI 并编写内容：

```bash
pnpm add -D @clarify-labs/cli
pnpm exec clarify init
pnpm dev
```

**clarify.json**

```json
{
  "title": "我的文档",
  "description": "这是 Clarify 驱动的文档站点"
}
```

然后在 `source` 目录下编写 `.md` / `.mdx` 文件（或放置 `.openapi.json` 规范），Clarify 会自动为你生成路由和页面。CLI 会自动装配开发服务器、客户端入口和静态构建流程。

- **开源项目维护者** — 需要为项目编写文档，希望文档和代码在同一个仓库

- **团队知识库** — 快速搭建内部文档站点，统一导航、搜索和品牌主色
