# 插件 API 参考

本文是 Clarify 插件机制的精确 API 说明。概念和使用场景请先阅读 [插件机制](/features/plugins)。

---

## `ClarifyPlugin`

```ts
type ClarifyPlugin = {
  name: string
  hooks: Partial<ClarifyHooks>
  slots?: UISlotRegistration[]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 插件名称，用于错误提示和调试 |
| `hooks` | `Partial<ClarifyHooks>` | 是 | 插件实现的生命周期 Hook |
| `slots` | `UISlotRegistration[]` | 否 | 注册的 UI 插槽组件 |

---

## UI 插槽

Clarify 支持通过插件在特定位置注入自定义 UI 组件。

### `UISlotName`

可用的插槽位置：

```ts
type UISlotName = 
  | 'page.footer.before'       // 页脚之前
  | 'page.banner.replace'      // 替换顶部横幅
  | 'page.footer.replace'      // 替换页脚
```

### `UISlotRegistration`

注册一个插槽组件：

```ts
type UISlotRegistration = {
  name: UISlotName      // 目标插槽位置
  component: string            // React 组件模块路径
}
```

### `useSlot`

插槽组件内部使用此 Hook 获取上下文：

```ts
import { useSlot } from 'virtual:clarify/slot'

function MySlotComponent() {
  const {
    name,       // 当前插槽名称
    plugin,     // 注册该插槽的插件名称
    route,      // 当前路由信息（如有）
    locale,     // 当前语言（如 en-US 或 zh-CN）
    DefaultComponent  // 仅替换插槽可用：原默认组件
  } = useSlot()
  // ...
}
```

### 示例：注册扩展插槽

```ts
// my-plugin.ts
export default {
  name: 'my-plugin',
  hooks: {},
  slots: [
    { 
      name: 'page.footer.before', 
      component: './components/MyFooterExtra.tsx' 
    }
  ]
}
```

### 示例：替换横幅

```ts
// my-plugin.ts
export default {
  name: 'my-theme',
  hooks: {},
  slots: [
    { 
      name: 'page.banner.replace', 
      component: './components/CustomBanner.tsx' 
    }
  ]
}
```

```tsx
// components/CustomBanner.tsx
import { useSlot, useConfig } from 'virtual:clarify/slot'

export default function CustomBanner() {
  const config = useConfig()
  const { DefaultComponent } = useSlot()
  
  // 可选：仍渲染原始横幅
  // return <DefaultComponent />
  
  return (
    <div className="my-custom-banner">
      欢迎使用 {config.title}！
    </div>
  )
}
```

## `ClarifyHooks`

```ts
type ClarifyHooks = {
  'routes:resolved'?: (
    input: { routes: ContentRoute[]; navigation: NavigationTree },
    ctx: ClarifyHookContext,
  ) =>
    | Promise<{ routes: ContentRoute[]; navigation: NavigationTree }>
    | { routes: ContentRoute[]; navigation: NavigationTree }

  'modules:before'?: (
    modules: Map<string, string>,
    ctx: ClarifyHookContext,
  ) => Promise<Map<string, string>> | Map<string, string>

  'build:done'?: (ctx: ClarifyHookContext) => Promise<void> | void
}
```

Hook 可以同步或异步返回。`routes:resolved` 和 `modules:before` 需要返回与输入同类型的结果，`build:done` 无需返回值。

## `ClarifyHookContext`

```ts
type ClarifyHookContext = {
  projectConfig: ResolvedProjectConfig
  generateOptions: ResolvedBuildOptions
}
```

### `projectConfig`

已应用默认值后的站点配置：

```ts
type ResolvedProjectConfig = {
  title: string
  logo?: string | { light?: string; dark?: string }
  favicon?: string | { light?: string; dark?: string }
  theme: {
    preset: 'default' | 'base'
    tokens: ResolvedClarifyThemeTokensConfig
    layout: ResolvedClarifyThemeLayoutConfig
  }
  description: string
  routePrefix: string
  navbar?: { links?: ClarifyNavbarLink[] }
  banner?: ClarifyBannerConfig
  footer?: ClarifyFooterConfig
  i18n?: ResolvedClarifyI18nConfig
  tabs?: ClarifyTabsConfig
}
```

### `generateOptions`

```ts
type ResolvedBuildOptions = {
  rootDirectory: string
  outputDirectory?: string
  ssg: {
    failOnError: boolean
  }
}
```

## 页面类型

```ts
type ClarifyPage = {
  path: string
  filePath: string
  frontmatter: Record<string, unknown>
  content: string
}
```

| 字段 | 说明 |
|------|------|
| `path` | 页面 URL 路径 |
| `filePath` | 内容文件绝对路径 |
| `frontmatter` | 解析后的页面元数据 |
| `content` | 页面正文内容 |

## 路由类型

```ts
type ContentRoute = {
  path: string
  basePath?: string
  locale?: string
  isFallback?: boolean
  alternates?: Record<string, string>
  title: string
  filePath: string
  virtualModuleId: string
  kind: 'mdx' | 'openapi'
  sections?: ContentSection[]
  rawContentUrl?: string
}
```

| 字段 | 说明 |
|------|------|
| `path` | 实际访问路径，启用 i18n 时可能带 locale 前缀 |
| `basePath` | 不带 locale 的基础路径 |
| `locale` | 当前路由所属语言 |
| `isFallback` | 是否由默认语言回退生成 |
| `alternates` | 同一页面在其他语言下的路径 |
| `title` | 导航和搜索展示标题 |
| `filePath` | 源文件绝对路径 |
| `virtualModuleId` | Clarify 内部虚拟模块 ID |
| `kind` | 页面类型：`mdx` 或 `openapi` |
| `sections` | 从 H2/H3 或 OpenAPI operation 中提取的页面目录 |
| `rawContentUrl` | 原始内容文件输出 URL |

## 导航类型

```ts
type NavigationTree =
  | ClarifyNavigationNode[]
  | LocalizedNavigation
  | TabbedNavigation
  | LocalizedTabbedNavigation

type LocalizedNavigation = Record<string, ClarifyNavigationNode[]>

type TabbedNavigation = { tabs: ClarifyNavigationTab[] }

type LocalizedTabbedNavigation = Record<string, TabbedNavigation>

type ClarifyNavigationTab = {
  type: 'tab'
  path: string
  title: string
  icon?: string
  children: ClarifyNavigationNode[]
}

type ClarifyNavigationNode = {
  path: string
  title: string
  icon?: string
  children?: ClarifyNavigationNode[]
  sections?: Pick<ContentSection, 'id' | 'title' | 'badge' | 'tags'>[]
}
```

Clarify 始终基于 `tabs` 配置构建导航，因此 Hook 收到的 `navigation` 是：

- 默认是 `TabbedNavigation`（`{ tabs }` 结构）。
- 启用 i18n 时按 locale 分组，为 `LocalizedTabbedNavigation`（`Record<string, { tabs }>`）。

`NavigationTree` 联合中的 `ClarifyNavigationNode[]` 和 `LocalizedNavigation` 是无 `tabs` 配置时的回退形态。

## 虚拟模块

`modules:before` 收到的是 `Map<string, string>`，键为虚拟模块 ID，值为模块源码。

常见虚拟模块包括：

| 模块 ID | 说明 |
|---------|------|
| `virtual:clarify-config` | 运行时站点配置 |
| `virtual:clarify-routes` | 路由和导航数据 |
| `virtual:clarify-openapi-registry` | 已解析的 OpenAPI 规范表 |
| `virtual:clarify-entry-client` | 客户端入口 |

修改虚拟模块属于高级用法，适合实现搜索、分析或内部平台集成等场景。
