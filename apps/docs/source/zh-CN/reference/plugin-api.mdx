# 插件 API 参考

本文是 Clarify 插件机制的精确 API 说明。概念和使用场景请先阅读 [插件机制](/features/plugins)。

---

## `ClarifyPlugin`

```ts
type ClarifyPlugin = {
  name: string
  enforce?: 'pre' | 'post'
  priority?: number
  dependsOn?: string[]
  hooks?: Partial<ClarifyHooks>
  slots?: UISlotRegistration[]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 插件名称，用于错误提示和调试 |
| `enforce` | `'pre' \| 'post'` | 否 | 将插件排在默认优先级插件之前或之后 |
| `priority` | `number` | 否 | 同一 enforce 分组内的数字排序 |
| `dependsOn` | `string[]` | 否 | 必须先于当前插件运行的插件名称 |
| `hooks` | `Partial<ClarifyHooks>` | 否 | 插件实现的生命周期 Hook |
| `slots` | `UISlotRegistration[]` | 否 | 注册的 UI 插槽组件 |

## UI 插槽

Clarify 支持通过插件在特定位置注入自定义 UI 组件。

### `UISlotName`

可用的插槽位置：

```ts
type UISlotName = 
  | 'page.footer.before'       // 页脚之前
  | 'page.banner.replace'      // 替换顶部横幅
  | 'page.footer.replace'      // 替换页脚
```

### `UISlotRegistration`

注册一个插槽组件：

```ts
type UISlotRegistration = {
  name: UISlotName
  component: string
}
```

`component` 是默认导出 React 组件的模块路径。使用 `/` 前缀表示从项目根目录解析，例如 `/source/plugins/MyFooter.tsx`。

### `useSlot`

插槽组件内部使用此 Hook 获取上下文：

```ts
import { useSlot } from 'virtual:clarify/slot'

function MySlotComponent() {
  const {
    name,       // 当前插槽名称
    plugin,     // 注册该插槽的插件名称
    route,      // 当前路由信息（如有）
    locale,     // 当前语言（如 en-US 或 zh-CN）
    DefaultComponent  // 仅替换插槽可用：原默认组件
  } = useSlot()
  // ...
}
```

### 示例：注册扩展插槽

```ts
// my-plugin.ts
export default {
  name: 'my-plugin',
  hooks: {},
  slots: [
    { 
      name: 'page.footer.before', 
      component: '/source/plugins/MyFooterExtra.tsx' 
    }
  ]
}
```

### 示例：替换横幅

```ts
// my-plugin.ts
export default {
  name: 'my-theme',
  hooks: {},
  slots: [
    { 
      name: 'page.banner.replace', 
      component: '/source/plugins/CustomBanner.tsx' 
    }
  ]
}
```

```tsx
// source/plugins/CustomBanner.tsx
import { useSlot } from 'virtual:clarify/slot'

export default function CustomBanner() {
  const { DefaultComponent } = useSlot()
  
  // 可选：仍渲染原始横幅
  // return <DefaultComponent />
  
  return (
    <div className="my-custom-banner">
      欢迎使用 Clarify！
    </div>
  )
}
```

## `ClarifyHooks`

```ts
type ClarifyHooks = {
  'before:config:load'?: ClarifyTapHook
  'after:config:load'?: ClarifyTapHook
  'before:config:resolve'?: ClarifyTapHook
  'after:config:resolve'?: ClarifyTapHook
  'before:plugins:load'?: ClarifyTapHook
  'after:plugins:load'?: ClarifyTapHook
  'before:site:discover'?: ClarifyTapHook
  'after:site:discover'?: ClarifyTapHook
  'before:content:process'?: ClarifyTapHook
  'after:content:process'?: ClarifyTapHook
  'before:modules:build'?: ClarifyTapHook
  'after:modules:build'?: ClarifyTapHook
  'before:build'?: ClarifyTapHook
  'after:build'?: ClarifyTapHook
  'before:ssg'?: ClarifyTapHook
  'after:ssg'?: ClarifyTapHook
  'before:dev:server'?: ClarifyTapHook
  'after:dev:server'?: ClarifyTapHook

  'content:transform'?: ClarifyPipelineHook<ClarifyContentTransformInput>

  'pages:resolved'?: (
    pages: ClarifyPage[],
    ctx: ClarifyHookContext,
  ) => Promise<ClarifyPage[]> | ClarifyPage[]

  'routes:discover'?: ClarifyPipelineHook<ClarifyRouteDiscoveryInput>
  'routes:discovered'?: ClarifyPipelineHook<ContentRoute[]>

  'routes:resolved'?: (
    input: { routes: ContentRoute[]; navigation: NavigationTree },
    ctx: ClarifyHookContext,
  ) =>
    | Promise<{ routes: ContentRoute[]; navigation: NavigationTree }>
    | { routes: ContentRoute[]; navigation: NavigationTree }

  'modules:before'?: (
    modules: Map<string, string>,
    ctx: ClarifyHookContext,
  ) => Promise<Map<string, string>> | Map<string, string>

  'html:transform'?: ClarifyPipelineHook<ClarifyHtmlTransformInput>
  'build:assets'?: ClarifyCollectorHook<ClarifyEmitAsset>
  'build:done'?: (ctx: ClarifyHookContext) => Promise<void> | void
  'build:shouldRun'?: ClarifyInterceptHook
  'ssg:shouldRun'?: ClarifyInterceptHook
  'dev:configureServer'?: (server: ViteDevServer, ctx: ClarifyHookContext) => Promise<void> | void
}
```

Pipeline hook 可以同步或异步返回，并且需要返回与输入相同形状的结果。Collector hook 返回额外生成项，tap hook 用于执行副作用，intercept hook 可以阻止某个构建阶段继续执行。

## `ClarifyHookContext`

```ts
type ClarifyHookContext = {
  version: string
  projectRoot: string
  contentRoot: string
  projectConfig: ResolvedProjectConfig
  generateOptions: ResolvedBuildOptions
  routes: ContentRoute[]
  navigation: NavigationTree
  plugins: ClarifyPlugin[]
  get<T>(key: string): T | undefined
  set<T>(key: string, value: T): void
  has(key: string): boolean
  delete(key: string): boolean
}
```

该上下文合并了项目路径、解析后的配置、当前路由/导航状态、已加载插件，以及可在 hook 间共享数据的小型插件 store。

| 字段 | 说明 |
|------|------|
| `version` | 当前构建使用的 Clarify 运行时版本 |
| `projectRoot` | 用户项目根目录的绝对路径 |
| `contentRoot` | 解析后的内容根目录绝对路径 |
| `projectConfig` | 已应用默认值后的站点配置 |
| `generateOptions` | 解析后的构建选项 |
| `routes` | 当前 hook 执行点的路由列表 |
| `navigation` | 当前 hook 执行点的导航树 |
| `plugins` | 排序完成后的已加载插件 |
| `get` / `set` / `has` / `delete` | 共享插件 store 辅助方法 |

### `projectConfig`

已应用默认值后的站点配置：

```ts
type ResolvedProjectConfig = {
  title: string
  logo?: string | { light?: string; dark?: string }
  favicon?: string | { light?: string; dark?: string }
  theme: {
    preset: 'default' | 'base'
    tokens: ResolvedClarifyThemeTokensConfig
    layout: ResolvedClarifyThemeLayoutConfig
  }
  description: string
  routePrefix: string
  navbar?: { links?: ClarifyNavbarLink[] }
  banner?: ClarifyBannerConfig
  footer?: ClarifyFooterConfig
  i18n?: ResolvedClarifyI18nConfig
  tabs?: ClarifyTabsConfig
}
```

### `generateOptions`

```ts
type ResolvedBuildOptions = {
  rootDirectory: string
  outputDirectory?: string
  ssg: {
    failOnError: boolean
  }
}
```

## 页面类型

```ts
type ClarifyPage = {
  path: string
  filePath: string
  frontmatter: Record<string, unknown>
  content: string
}
```

| 字段 | 说明 |
|------|------|
| `path` | 页面 URL 路径 |
| `filePath` | 内容文件绝对路径 |
| `frontmatter` | 解析后的页面元数据 |
| `content` | 页面正文内容 |

## 路由类型

```ts
type ContentRoute = {
  path: string
  basePath?: string
  locale?: string
  isFallback?: boolean
  isBareAlias?: boolean
  alternates?: Record<string, string>
  kind: string
  meta: ContentRouteMeta
  module: ContentRouteModule
  source: ContentRouteSource
  openapi?: OpenAPIContentRouteState
  diagnostic?: ContentDiagnostic
  artifacts?: ContentRouteArtifacts
}

type ContentRouteMeta = {
  title: string
  description?: string
  keywords?: string[]
  sections?: ContentSection[]
}

type ContentRouteModule = {
  virtualModuleId: string
}

type ContentRouteSource = {
  filePath: string
  frontmatter?: Record<string, unknown>
  content?: string
  sourceEditUrl?: string
}

type ContentRouteArtifacts = {
  contentArtifactUrl?: string
}

type OpenAPIContentRouteState = {
  tagFilter?: string[]
  sourceSpecId?: string
  routeSpecModuleId?: string
}
```

| 字段 | 说明 |
|------|------|
| `path` | 实际访问路径，启用 i18n 时可能带 locale 前缀 |
| `basePath` | 不带 locale 的基础路径 |
| `locale` | 当前路由所属语言 |
| `isFallback` | 是否由默认语言回退生成 |
| `isBareAlias` | 是否是默认语言的非索引别名路由 |
| `alternates` | 同一页面在其他语言下的路径 |
| `kind` | 路由类型，例如 `mdx` 或 `openapi` |
| `meta` | 标题、描述、关键词和页面目录等展示元数据 |
| `module` | 当前路由的运行时模块信息 |
| `source` | 源文件、frontmatter、标准化内容和编辑链接信息 |
| `openapi` | OpenAPI 专属状态，仅 OpenAPI 路由存在 |
| `diagnostic` | 诊断路由的可选诊断信息 |
| `artifacts` | 当前路由生成的内容产物 URL |

## 导航类型

```ts
type NavigationTree =
  | FlatNavigationTree
  | LocalizedNavigation
  | TabbedNavigation
  | LocalizedTabbedNavigation

type FlatNavigationTree = {
  kind: 'flat'
  nodes: ClarifyNavigationNode[]
}

type LocalizedNavigation = {
  kind: 'localized'
  locales: Record<string, ClarifyNavigationNode[]>
}

type TabbedNavigation = {
  kind: 'tabbed'
  tabs: ClarifyNavigationTab[]
}

type LocalizedTabbedNavigation = {
  kind: 'localized-tabbed'
  locales: Record<string, { tabs: ClarifyNavigationTab[] }>
}

type ClarifyNavigationTab = {
  type: 'tab'
  path: string
  title: string
  icon?: string
  children: ClarifyNavigationNode[]
}

type ClarifyNavigationNode = {
  path: string
  title: string
  icon?: string
  children?: ClarifyNavigationNode[]
  sections?: NavigationSection[]
}

type NavigationSection = {
  id: string
  title: string
  badge?: string
  tags?: string[]
}
```

`NavigationTree` 始终使用 `kind` 作为判别字段。`flat` 和 `localized` 表示侧边栏导航，`tabbed` 和 `localized-tabbed` 表示从 `tabs` 配置生成的导航。

## 虚拟模块

`modules:before` 收到的是 `Map<string, string>`，键为虚拟模块 ID，值为模块源码。

常见虚拟模块包括：

| 模块 ID | 说明 |
|---------|------|
| `virtual:clarify/config` | 运行时站点配置 |
| `virtual:clarify/routes` | 路由和导航数据 |
| `virtual:clarify/openapi` | 已解析的 OpenAPI 规范表 |
| `virtual:clarify/entry-client` | 客户端入口 |

修改虚拟模块属于高级用法，适合实现搜索、分析或内部平台集成等场景。
