# 内置公开组件

Clarify 在 MDX 中默认注册了一组公开组件。你无需手动 `import`，可直接在 `.mdx` 文件里使用它们来组织首页入口、提示信息、操作按钮和代码示例。

---

## 自动可用的 MDX 组件

以下组件可直接在 MDX 页面中使用：

- 内容提示：`Callout`、`Note`
- 入口与动作：`CardGroup` / `Card`、`Button`
- 代码与折叠：`CodeGroup`、`Collapse`
- 页面组织：`Steps` / `Step`、`Tabs` / `Tab`
- 布局与属性说明：`Row` / `Col`、`Properties` / `Property`
- 内容嵌入：`WebFrame`
- Markdown 渲染：`Markdown`
- OpenAPI：`OpenApiOperation`、`OpenApiDocument`

## Callout

`Callout` 适合放置与正文强相关的说明、注意事项或风险提示。

### 基础用法

```mdx
<Callout type="info">
  模型列表会持续更新，生产接入前应以控制台或模型广场中当前可用模型为准。
</Callout>
```

### 类型

`type` 支持 `info`、`note`、`tip`、`warning`、`danger`、`success`。

<Callout type="note" title="Note">
  适合补充说明上下文，避免正文被细节打断。
</Callout>

<Callout type="tip" title="Tip">
  适合展示推荐实践、效率技巧或更优路径。
</Callout>

<Callout type="warning" title="Warning">
  适合展示可能导致构建失败、权限不足或行为变化的注意事项。
</Callout>

<Callout type="danger" title="Danger">
  适合展示高风险操作，例如删除数据、重置环境或暴露密钥。
</Callout>

<Callout type="success" title="Success">
  适合展示操作完成、配置生效或验证通过。
</Callout>

### 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `type` | `'info' \| 'note' \| 'tip' \| 'warning' \| 'danger' \| 'success'` | `'info'` | 控制颜色和图标 |
| `title` | `ReactNode` | 根据 `type` 自动生成 | 标题，传空值可隐藏 |
| `icon` | `boolean` | `true` | 是否显示图标 |
| `className` | `string` | - | 追加样式类名 |

## Note

`Note` 是更轻量的内联提示组件，适合放置不会改变任务流程的补充说明。和 `Callout` 相比，它不需要选择类型，也没有标题栏。

```mdx
<Note>
  标题、H2/H3 和导航命名会影响站内搜索结果，建议使用用户能理解的词。
</Note>
```

<Note>
  标题、H2/H3 和导航命名会影响站内搜索结果，建议使用用户能理解的词。
</Note>

### 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `children` | `ReactNode` | 必填 | 提示内容 |

## CardGroup / Card

`CardGroup` 和 `Card` 适合做文档入口、功能清单、方案矩阵或章节导航。

### 基础用法

```mdx
<CardGroup cols={2}>
  <Card title="关于 Clarify" icon="badge-info" href="/getting-started">
    了解 Clarify 的产品定位、核心价值和推荐使用路径。
  </Card>
  <Card title="快速开始" icon="rocket" href="/getting-started">
    准备环境、安装 CLI 并创建第一个文档站点。
  </Card>
</CardGroup>
```

渲染效果：

<CardGroup cols={2}>
  <Card title="关于 Clarify" icon="badge-info" href="/getting-started">
    了解 Clarify 的产品定位、核心价值和推荐使用路径。
  </Card>
  <Card title="快速开始" icon="rocket" href="/getting-started">
    准备环境、安装 CLI 并创建第一个文档站点。
  </Card>
  <Card title="写作文档" icon="file-code-2" href="/guides/writing-content">
    使用 Markdown、JSX 和内置组件编写更丰富的内容。
  </Card>
  <Card title="API 文档" icon="braces" href="/features/openapi">
    从 OpenAPI 描述文件生成接口文档和单接口展示。
  </Card>
</CardGroup>

### 图标命名

`Card` 的 `icon` 使用 `lucide-react` 图标名，支持 kebab-case 自动转换。例如：

| 写法 | 对应图标 |
|------|----------|
| `badge-info` | `BadgeInfo` |
| `rocket` | `Rocket` |
| `file-code-2` | `FileCode2` |
| `shield-check` | `ShieldCheck` |

### 属性

#### CardGroup

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `cols` | `1 \| 2 \| 3 \| 4` | `2` | 桌面端列数 |
| `className` | `string` | - | 追加样式类名 |

#### Card

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | `string` | 必填 | 卡片标题 |
| `icon` | `string` | - | lucide 图标名，支持 kebab-case |
| `href` | `string` | - | 卡片跳转链接，站内路径会使用客户端路由 |
| `className` | `string` | - | 追加样式类名 |

## Collapse

`Collapse` 适合把可选补充说明、实现细节或长列表内容收起，保持文档主流程简洁。

```mdx
<Collapse title="查看详细说明">
  这是默认折叠的内容，只有在用户点击标题后才会显示。
</Collapse>

<Collapse summary="默认展开" defaultOpen>
  这段内容初始可见，用户仍然可以手动收起。
</Collapse>
```

<Collapse title="查看详细说明">
  这是默认折叠的内容，只有在用户点击标题后才会显示。
</Collapse>

### 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | `ReactNode` | - | 折叠面板标题，优先级高于 `summary` |
| `summary` | `ReactNode` | `'Details'` | 备用标题 |
| `defaultOpen` | `boolean` | `false` | 是否默认展开 |
| `className` | `string` | - | 追加样式类名 |

## Button

`Button` 适合在文档中提供明确的下一步动作。

```mdx
<Button href="/getting-started" arrow="right">
  开始安装
</Button>

<Button href="https://github.com/taicode-labs/clarify" variant="outline">
  查看 GitHub
</Button>
```

渲染效果：

<div className="not-prose my-6 flex flex-wrap gap-3">
  <Button href="/getting-started" arrow="right">
    开始安装
  </Button>
  <Button href="https://github.com/taicode-labs/clarify" variant="outline">
    查看 GitHub
  </Button>
</div>

### 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `variant` | `'primary' \| 'secondary' \| 'filled' \| 'outline' \| 'text'` | `'primary'` | 按钮样式 |
| `arrow` | `'left' \| 'right'` | - | 是否显示方向箭头 |
| `href` | `string` | - | 存在时渲染为链接,否则渲染为按钮 |

## Steps / Step

`Steps` 和 `Step` 用于引导用户完成带编号的顺序步骤。它们非常适合安装指南、设置说明、教程和任何多步骤流程。

### 基础用法

```mdx
<Steps>
  <Step title="安装依赖">
    运行以下命令安装 Clarify CLI：
    `pnpm add -D @clarify-labs/cli`
  </Step>

  <Step title="创建配置文件">
    在项目根目录创建 `clarify.ts` 文件并配置。
  </Step>

  <Step title="启动开发服务器">
    运行开发服务器预览文档：
    `pnpm exec clarify dev`
  </Step>
</Steps>
```

渲染效果：

<Steps>
  <Step title="安装依赖">
    运行以下命令安装 Clarify CLI：`pnpm add -D @clarify-labs/cli`
  </Step>
  <Step title="创建配置文件">
    在项目根目录创建 `clarify.ts` 文件，配置文档标题和导航结构。
  </Step>
  <Step title="启动开发服务器">
    运行开发服务器预览文档：`pnpm exec clarify dev`
  </Step>
</Steps>

### 自定义图标

你可以使用 Lucide 图标名称（kebab-case 或 PascalCase）作为步骤指示器：

```mdx
<Steps>
  <Step title="下载" icon="download">
    下载安装包。
  </Step>
  <Step title="安装" icon="package">
    运行安装程序并按照提示操作。
  </Step>
  <Step title="配置" icon="settings">
    设置你的偏好和 API 密钥。
  </Step>
</Steps>
```

<Steps>
  <Step title="下载" icon="download">
    下载安装包。
  </Step>
  <Step title="安装" icon="package">
    运行安装程序并按照提示操作。
  </Step>
  <Step title="配置" icon="settings">

### 自定义编号

你可以使用 `stepNumber` 属性覆盖自动编号：

```mdx
<Steps>
  <Step title="前置条件" stepNumber={0}>
    在安装前检查系统要求。
  </Step>
  <Step title="安装">
    安装软件包。
  </Step>
  <Step title="验证">
    验证安装是否成功。
  </Step>
</Steps>
```

### 属性

#### Steps

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `titleSize` | `'p' \| 'h2' \| 'h3' \| 'h4'` | `'h3'` | 步骤标题的 HTML 标签 |
| `className` | `string` | - | 额外的 CSS 类名 |

#### Step

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | `string` | 必需 | 步骤标题 |
| `icon` | `string \| ReactNode` | - | Lucide 图标名称或自定义 React 元素 |
| `stepNumber` | `number` | 自动编号 | 覆盖自动编号 |
| `id` | `string` | 自动生成 | 用于锚点链接的 HTML id；如果未提供则从标题自动生成 |
| `noAnchor` | `boolean` | `false` | 隐藏锚点链接图标 |
| `titleSize` | `'p' \| 'h2' \| 'h3' \| 'h4'` | `'h3'` | 此步骤标题的 HTML 标签；未设置时继承自 `Steps` |
| `className` | `string` | - | 额外的 CSS 类名 |

## Tabs / Tab

`Tabs` 和 `Tab` 用于将相关内容组织在可切换的面板中。它们非常适合展示不同语言的代码示例、不同平台的配置或同一任务的不同方法。

### 基础用法

```mdx
<Tabs>
  <Tab title="npm">
    安装：`npm install @clarify-labs/cli`
  </Tab>
  <Tab title="pnpm">
    安装：`pnpm add @clarify-labs/cli`
  </Tab>
  <Tab title="yarn">
    安装：`yarn add @clarify-labs/cli`
  </Tab>
</Tabs>
```

渲染效果：

<Tabs>
  <Tab title="npm">
    使用 npm 安装：`npm install @clarify-labs/cli`
  </Tab>
  <Tab title="pnpm">
    使用 pnpm 安装：`pnpm add @clarify-labs/cli`
  </Tab>
  <Tab title="yarn">
    使用 yarn 安装：`yarn add @clarify-labs/cli`
  </Tab>
</Tabs>

### 带图标

使用 Lucide 图标名称为标签添加视觉指示器：

```mdx
<Tabs>
  <Tab title="JavaScript" icon="javascript">
    示例：`export default function hello() { console.log('Hello, world!') }`
  </Tab>
  <Tab title="TypeScript" icon="braces">
    示例：`export default function hello(): void { console.log('Hello, world!') }`
  </Tab>
  <Tab title="Python" icon="code">
    示例：`def hello(): print('Hello, world!')`
  </Tab>
</Tabs>
```

<Tabs>
  <Tab title="JavaScript" icon="javascript">
    JavaScript 示例：`export default function hello() { console.log('Hello, world!') }`
  </Tab>
  <Tab title="TypeScript" icon="braces">
    TypeScript 示例：`export default function hello(): void { console.log('Hello, world!') }`
  </Tab>
  <Tab title="Python" icon="code">
    Python 示例：`def hello(): print('Hello, world!')`
  </Tab>

### 同步标签

默认情况下，页面中具有相同标题的标签会自动同步。当用户选择一个标签时，所有具有匹配标题的其他标签组都会自动更新。这对于在整个文档中保持一致的语言或平台选择非常有用。

```mdx
<Tabs>
  <Tab title="macOS">
    macOS 用户说明。
  </Tab>
  <Tab title="Windows">
    Windows 用户说明。
  </Tab>
  <Tab title="Linux">
    Linux 用户说明。
  </Tab>
</Tabs>

稍后在同一页面：

<Tabs>
  <Tab title="macOS">
    额外的 macOS 专属信息。
  </Tab>
  <Tab title="Windows">
    额外的 Windows 专属信息。
  </Tab>
  <Tab title="Linux">
    额外的 Linux 专属信息。
  </Tab>
</Tabs>
```

<Callout type="tip">
  当用户在第一个标签组中点击"Windows"时，第二个标签组也会自动切换到"Windows"。要禁用此行为，请设置 `sync={false}`。
</Callout>

### 属性

#### Tabs

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `defaultTabIndex` | `number` | `0` | 初始选中的标签索引 |
| `sync` | `boolean` | `true` | 是否在所有具有匹配标题的标签组之间同步选择 |
| `borderBottom` | `boolean` | `true` | 在标签列表下方添加底部边框 |
| `className` | `string` | - | 额外的 CSS 类名 |

#### Tab

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | `string` | 必需 | 标签标签 |
| `icon` | `string \| ReactNode` | - | Lucide 图标名称或自定义 React 元素 |
| `id` | `string` | - | 标签面板的 HTML id |
| `className` | `string` | - | 额外的 CSS 类名 |

## WebFrame

`WebFrame` 用于在文档页面中嵌入网页内容，例如产品界面、托管演示、生成预览或其他站内页面。默认会渲染模拟浏览器窗口；如果不需要窗口外壳，可以设置 `showWindow={false}`。

### 基础用法

```mdx
<WebFrame src="https://clarify.pub" title="主题预览" />
```

渲染效果：

<WebFrame src="https://clarify.pub" title="主题预览" height={360} />

### 不显示模拟窗口

```mdx
<WebFrame src="https://clarify.pub" title="组件预览" showWindow={false} height={320} />
```

### 绝对 URL

嵌入外部页面时可以使用带协议的完整 URL。

```mdx
<WebFrame src="https://clarify.pub" title="Clarify 官网" height={320} />
```

渲染效果：

<WebFrame src="https://clarify.pub" title="Clarify 官网" height={320} />

### 内联内容

如果不传 `src`，`WebFrame` 会在框架中渲染 `children`，适合小型产品 UI mockup 或静态预览。

```mdx
<WebFrame title="内联预览" height={180}>
  <div style={{ padding: 24 }}>
    <strong>预览内容</strong>
    <p>这段内容会直接渲染在框架内部。</p>
  </div>
</WebFrame>
```

### 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `src` | `string` | - | 通过 iframe 嵌入的 URL |
| `title` | `string` | `'Embedded content'` | iframe 可访问标题和模拟窗口标签 |
| `showWindow` | `boolean` | `true` | 是否渲染模拟浏览器窗口外壳 |
| `height` | `CSSProperties['height']` | `420` | 嵌入内容高度 |
| `className` | `string` | - | 外层容器追加样式类名 |
| `iframeClassName` | `string` | - | iframe 或内联内容容器追加样式类名 |
| `children` | `ReactNode` | - | 未提供 `src` 时渲染的内联内容 |

## Row / Col

`Row` 和 `Col` 用于在大屏上创建左右两栏布局，常见于“左侧解释、右侧示例”或“左侧步骤、右侧接口”的文档结构。

#### Row

|------|------|--------|------|
| `children` | `ReactNode` | 必填 | 当前列内容 |
| `sticky` | `boolean` | `false` | 大屏下是否吸顶 |

## OpenAPI 组件

`OpenApiOperation` 用于在指南中嵌入单个接口，`OpenApiDocument` 用于嵌入完整 API Reference。

```mdx
<OpenApiOperation specPath="/api" path="/sessions" method="post" />

<OpenApiDocument specPath="/api" />
```

| 组件 | 关键属性 | 说明 |
|------|----------|------|
| `OpenApiOperation` | `specPath`、`path`、`method` | 展示单个接口，适合任务型指南 |
| `OpenApiDocument` | `specPath` | 展示完整 OpenAPI Reference |

OpenAPI 面板会根据规范展示服务器、认证、参数、请求体、响应示例，并生成多语言请求代码。更多说明见 [API 文档](/features/openapi)。

## 使用建议

- 首页、章节索引和功能矩阵优先使用 `CardGroup` / `Card`。
- 与正文强相关的限制、风险和补充说明使用 `Callout`，轻量补充说明使用 `Note`。
- 引导用户继续阅读或执行操作时使用 `Button`。
- 顺序任务、安装指南、教程和多步骤流程使用 `Steps` / `Step`。
- 不同语言的代码示例、特定平台的说明或替代方法使用 `Tabs` / `Tab`；利用 `sync={true}` 在整个页面保持一致的选择。
- 多语言、多环境、多包管理器示例使用 `CodeGroup`。
- 可选细节、长列表和进阶内容使用 `Collapse`。
- 嵌入演示、预览和产品界面时使用 `WebFrame`；如果被嵌入内容本身已有窗口外壳，可关闭 `showWindow`。
- 配置项、组件属性和字段说明优先使用 `Properties` / `Property`。
- 指南中解释流程，接口事实交给 `OpenApiOperation` 或 `OpenApiDocument`。

## 自定义 MDX 组件

如果你需要项目专属组件，可以在 MDX 文件顶部显式导入：

```mdx
import { Alert } from '../components/Alert'

<Alert type="warning">
  这是一个自定义 React 组件示例。
</Alert>
```

Clarify 的内置组件会自动在 MDX 中注册；自定义组件需要手动导入。
