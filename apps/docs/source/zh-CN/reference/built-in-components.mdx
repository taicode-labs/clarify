# 内置组件

Clarify 会自动在 MDX 中注册一组公开组件。你无需手动 `import`，可以直接使用它们组织流程、切换内容、展示提示、构建入口和嵌入预览。

## 自动注册

这些组件只在 Clarify 的 MDX 页面中自动可用。项目自定义 React 组件仍需在页面顶部显式导入：

```mdx
import { Status } from '../../components/Status'

<Status value="ready" />
```

<Callout type="tip">
  块级 Markdown、代码围栏和 JSX 组件之间保留空行，可以避免内容被解析为内联文本。
</Callout>

## 组合建议

- 用 `Steps` 表达顺序，用 `Tabs` 表达互斥选择，不要用它们模拟普通章节。
- 用 `Callout` 强调会影响决策的信息，用 `Note` 补充不影响主流程的背景。
- 用 `Card` 导航到另一页，用 `Button` 触发明确的下一步。
- 用 `CodeGroup` 比较等价代码，用 `Collapse` 隐藏非必读细节。
- 用 `AccordionGroup` 组织相关折叠项，用 `FileTree` 展示目录结构，用 `Figure` 展示带图注的图片。
- `Tooltip` 只适合简短的补充定义；必须阅读的信息应直接写在页面中。
- 用 `Mermaid` 绘制需要版本管理且可搜索的技术图示。
- 数据驱动的 Markdown 字符串交给 `Markdown`；直接写在 `.mdx` 中的内容无需包裹。
