# Properties / Property

`Properties` 和 `Property` 用于描述配置项、组件 Props 和数据字段。它们比宽表格更适合长说明，并能在窄屏保持可读。

## 用法

```mdx
<Properties>
  <Property name="routePrefix" type="string">
    部署子路径，例如 `/docs/`。
  </Property>
  <Property name="tabs" type="ClarifyTabsConfig">
    配置顶部业务分类和侧边栏结构。
  </Property>
</Properties>
```

<Properties>
  <Property name="routePrefix" type="string">部署子路径，例如 `/docs/`。</Property>
  <Property name="tabs" type="ClarifyTabsConfig">配置顶部业务分类和侧边栏结构。</Property>
</Properties>

## 属性

### Properties

<Properties>
  <Property name="children" type="ReactNode">一个或多个 `Property`。</Property>
  <Property name="className" type="string">可选的外层容器类名。</Property>
  <Property name="spacingClassName" type="string">可选的间距类名。默认值为 `my-6`。</Property>
</Properties>

### Property

<Properties>
  <Property name="name" type="string">字段或属性名。</Property>
  <Property name="type" type="string">可选的类型说明。</Property>
  <Property name="children" type="ReactNode">详细说明，支持 MDX。</Property>
</Properties>

<Callout type="tip">短且可比较的数据仍适合表格；说明包含多段文字、链接或示例时使用 `Properties`。</Callout>
