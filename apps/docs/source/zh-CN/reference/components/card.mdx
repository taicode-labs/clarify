# CardGroup / Card

`CardGroup` 负责响应式网格，`Card` 提供带标题、图标和链接的内容入口。适合首页入口、章节索引和功能矩阵。

## 基础用法

```mdx
<CardGroup cols={2}>
  <Card title="快速开始" icon="rocket" href="/getting-started">
    安装 CLI 并创建第一个文档站点。
  </Card>
  <Card title="写作文档" icon="file-code-2" href="/guides/writing-content">
    使用 Markdown、JSX 和内置组件。
  </Card>
</CardGroup>
```

<CardGroup cols={2}>

  <Card title="写作文档" icon="file-code-2" href="/guides/writing-content">使用 Markdown、JSX 和内置组件。</Card>
</CardGroup>

## 图标与属性

`icon` 使用 `lucide-react` 图标名，支持 kebab-case 自动转换，例如 `badge-info` 和 `file-code-2`。

### CardGroup

<Properties>
  <Property name="cols" type="1 | 2 | 3 | 4">桌面端列数，窄屏自动堆叠。默认值为 `2`。</Property>
  <Property name="className" type="string">可选的网格类名。</Property>
</Properties>

### Card

<Properties>
  <Property name="title" type="string">必填的卡片标题。</Property>
  <Property name="icon" type="string">可选的 Lucide 图标名。</Property>
  <Property name="href" type="string">可选的站内或外部链接。</Property>
  <Property name="children" type="ReactNode">可选的简短描述。</Property>
  <Property name="className" type="string">可选的卡片类名。</Property>
</Properties>

<Callout type="warning">卡片应作为入口而不是正文容器。描述保持简短，详细内容放到目标页面。</Callout>
