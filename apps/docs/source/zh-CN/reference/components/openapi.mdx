# OpenAPI 组件

`OpenApiOperation` 在任务型指南中嵌入一个接口，`OpenApiDocument` 在页面中嵌入完整或按标签筛选的 API Reference。两者都会展示认证、参数、请求体、响应和请求示例。

## 单个接口

```mdx
<OpenApiOperation specPath="/api" operationId="demoBasicRequest" />
```

`operationId` 必须唯一标识规范中的 operation。

渲染效果：

<OpenApiOperation specPath="/api" operationId="demoBasicRequest" />

## 接口跳转

当指南只需要引导读者前往完整 API Reference，而不需要在当前页面展开完整接口详情时，可以使用 `OpenApiLink`：

```mdx
<OpenApiLink specPath="/api" operationId="demoBasicRequest" />
```

如果需要把链接放进段落中，可以设置 `inline`，使用更紧凑的行内样式：

```mdx
更多信息请参考 <OpenApiLink specPath="/api" operationId="demoBasicRequest" inline /> 的 API 文档。
```

更多信息请参考 <OpenApiLink specPath="/api" operationId="demoBasicRequest" inline /> 的 API 文档。

组件会以紧凑的接口样式展示 HTTP 方法、operation 摘要和路径，并自动链接到对应的 operation 锚点。如果规范注册路径和公开的 API Reference 路径不同，可以使用 `href`：

```mdx
<OpenApiLink specPath="/api" href="/reference/api" operationId="demoBasicRequest" />
```

<OpenApiLink specPath="/api" operationId="demoBasicRequest" />

接口栏中的播放按钮会打开交互式请求测试器。用户可以选择服务地址和认证方式，编辑参数与请求体，并查看状态码、耗时、响应头和响应体。

## 内嵌请求测试器

当发起请求本身就是指南流程的一部分时，可以直接嵌入同一个测试器：

```mdx
<OpenApiRequest specPath="/api" operationId="demoBasicRequest" requestExample="published" />
```

`requestExample` 按 key 选择初始 OpenAPI 请求体或参数示例；同名的参数和请求体示例会一起应用。

<OpenApiRequest specPath="/api" operationId="demoBasicRequest" requestExample="published" />

## 完整文档

```mdx
<OpenApiDocument specPath="/api" />
```

按标签限制展示范围：

```mdx
<OpenApiDocument specPath="/api" tagFilter={["Users", "Teams"]} />
```

## 属性

### OpenApiOperation

<Properties>
	<Property name="specPath" type="string">配置中的 OpenAPI 规范路径。</Property>
	<Property name="operationId" type="string">唯一的 OpenAPI operation 标识。</Property>
</Properties>

### OpenApiLink

<Properties>
	<Property name="specPath" type="string">配置中的 OpenAPI 规范路径。</Property>
	<Property name="operationId" type="string">唯一的 OpenAPI operation 标识。</Property>
	<Property name="href" type="string">可选的公开 API Reference 路径，默认使用 `specPath`。</Property>
	<Property name="inline" type="boolean">使用紧凑的行内链接，而不是 endpoint 卡片。默认值为 `false`。</Property>
	<Property name="className" type="string">可选的链接类名。</Property>
</Properties>

### OpenApiRequest

<Properties>
	<Property name="specPath" type="string">配置中的 OpenAPI 规范路径。</Property>
	<Property name="operationId" type="string">唯一的 OpenAPI operation 标识。</Property>
	<Property name="requestExample" type="string">初始请求体或参数示例的 key。</Property>
</Properties>

### OpenApiDocument

<Properties>
	<Property name="specPath" type="string">配置中的 OpenAPI 规范路径。</Property>
	<Property name="tagFilter" type="string[]">仅展示匹配标签的接口。</Property>
</Properties>

<Callout type="tip">指南负责解释任务和决策，把接口事实交给 OpenAPI 组件，避免手写参数表与规范漂移。</Callout>

更多配置方式见 [OpenAPI 文档](/features/openapi) 和 [在 MDX 中嵌入 OpenAPI](/openapi/embedding)。
