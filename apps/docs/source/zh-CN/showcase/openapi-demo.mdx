# API 文档示例

本页展示 Clarify 如何把同一份 OpenAPI 规范用于完整 Reference 和 MDX 接口嵌入。

示例规范位于 `source/zh-CN/api.openapi.json`，生成的完整页面是 [API Reference](/api)。

---

## 示例规范覆盖范围

当前示例是一个 OpenAPI 3.1 fixture，用来覆盖常见渲染场景：

- `info`、`servers`、`externalDocs`、`tags`
- 全局 `security` 和 `components.securitySchemes`
- path / query / header 参数
- JSON、YAML、multipart、merge-patch 等请求体
- 多状态码响应、headers、examples、links
- `allOf`、`oneOf`、enum、const、format、pattern、additionalProperties、discriminator
- callbacks、webhooks、OpenAPI 3.1 `jsonSchemaDialect`

---

## 接口目录预览

```txt
GET     /pages              List pages
POST    /pages              Create a page
OPTIONS /pages              Describe page collection capabilities
GET     /pages/{pageId}     Get a page
PUT     /pages/{pageId}     Replace a page
PATCH   /pages/{pageId}     Patch a page
DELETE  /pages/{pageId}     Delete a page
HEAD    /pages/{pageId}     Check if a page exists
POST    /assets             Upload an asset
GET     /search             Search pages
POST    /exports            Create an export
GET     /system/health      Get service health
TRACE   /system/health      Trace health request
```

---

## 示例一：列表接口

在指南中解释“读取页面列表”时，可以直接嵌入 `GET /pages`：

```mdx
<OpenApiEndpoint specPath="/api" path="/pages" method="get" />
```

渲染效果：

<OpenApiEndpoint specPath="/api" path="/pages" method="get" />

---

## 示例二：复杂请求体和多响应

`POST /exports` 覆盖更复杂的接口结构：请求体有 JSON 与 YAML 两种 media type，每种 media type 下都有多个 examples；响应包含 `202` 和 `400`，成功响应同时提供 JSON 与纯文本 media type。

```mdx
<OpenApiEndpoint specPath="/api" path="/exports" method="post" />
```

渲染效果：

<OpenApiEndpoint specPath="/api" path="/exports" method="post" />

---

## 原始规范

构建后，原始 OpenAPI 文件会输出到稳定 URL：

- [api.openapi.json](/api.openapi.json)

这个链接也会出现在 `llms.txt` 中，方便 AI 工具读取机器可解析的接口规范。
