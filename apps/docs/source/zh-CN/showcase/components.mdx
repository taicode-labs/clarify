## 内置公开组件

Clarify 默认提供一组可直接在 MDX 中使用的组件，不需要手动导入。完整 API 说明见 [内置公开组件](/reference/built-in-components)。

### Callout

`Callout` 用于强调说明、风险提示、推荐实践或操作结果。常见类型包括 `info`、`note`、`tip`、`warning`、`danger` 和 `success`。

```mdx
<Callout type="info">
  模型列表会持续更新，生产接入前应以控制台或模型广场中当前可用模型为准。
</Callout>
```

<Callout type="info">
  模型列表会持续更新，生产接入前应以控制台或模型广场中当前可用模型为准。
</Callout>

<Callout type="warning" title="注意">
  生产环境发布前，请确认密钥、域名、权限和缓存策略都已经完成校验。
</Callout>

<Callout type="success" title="已完成">
  构建通过后，Clarify 会为每个路由生成可直接部署的静态 HTML。
</Callout>

### CardGroup / Card

`CardGroup` 和 `Card` 用于创建章节入口、功能矩阵或首页导航。`Card` 的 `icon` 使用 lucide 图标名，支持 `badge-info` 这种 kebab-case 写法。

```mdx
<CardGroup cols={2}>
  <Card title="快速开始" icon="rocket" href="/getting-started">
    安装 CLI 并创建第一个 Clarify 文档站点。
  </Card>
  <Card title="写作文档" icon="file-code-2" href="/guides/writing-content">
    使用 Markdown、JSX 和内置组件组织内容。
  </Card>
</CardGroup>
```

### WebFrame

`WebFrame` 用于嵌入产品预览、在线演示或生成页面等网页内容。默认会显示模拟浏览器窗口。

```mdx
<WebFrame src="/showcase/theming-preview" title="主题预览" height={320} />

<WebFrame src="/showcase/components" title="组件预览" showWindow={false} height={280} />

<WebFrame src="https://clarify.pub" title="OpenStreetMap 嵌入" height={280} />
```

## 自定义 MDX 组件

你也可以在 Markdown 中直接使用 JSX 或导入自己的 React 组件：

<div className="bg-blue-50 border border-blue-200 p-4 rounded-lg dark:border-blue-400/30 dark:bg-blue-400/10">
  <h4 className="text-blue-800 font-semibold mb-2 dark:text-blue-200">提示</h4>
  <p className="text-blue-700 dark:text-blue-100">
    这是一个自定义 React 组件，直接在 MDX 中渲染！
  </p>
</div>
