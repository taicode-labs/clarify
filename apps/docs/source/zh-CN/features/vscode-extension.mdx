# VS Code 扩展

Clarify VS Code 扩展可以让你在 Visual Studio Code 里直接预览 Clarify 文档项目。

当你编写 Markdown、MDX 或 OpenAPI 文档时，可以用它在编辑器内打开当前页面对应的实时预览。

## 安装扩展

通过 Clarify Release 中提供的扩展包安装：

1. 从最新的 Clarify GitHub Release 下载 `clarify-vscode-extension-*.vsix` 文件。
2. 在 VS Code 中打开命令面板。
3. 运行 `Extensions: Install from VSIX...`。
4. 选择下载好的 `.vsix` 文件。
5. 如果 VS Code 提示，重新加载窗口。

安装完成后，扩展会在 Extensions 视图中显示为 **Clarify**。

## 打开预览

1. 在 VS Code 中打开 Clarify 项目文件夹。
2. 打开 `index.mdx`、`guide.mdx`、`api.openapi.json`、`api.openapi.yaml` 或 `api.openapi.yml` 等内容文件。
3. 从命令面板运行 `Clarify: Preview`，也可以点击编辑器标题栏中出现的 Clarify 图标。
4. 扩展会启动本地 Clarify dev server，并在 VS Code webview 面板中打开当前文件对应的文档页面。

预览默认会在当前编辑器旁边打开。切换到其他 Clarify 内容文件时，预览也可以自动跟随当前文件更新。

## 功能

- 识别当前工作区中的 Clarify 项目。
- 识别内容根目录内的 Markdown、MDX 和 OpenAPI 源文件。
- 为项目启动并复用本地 Clarify 预览服务。
- 根据正在编辑的文件解析对应路由并在预览中打开。
- 在内容、配置或依赖变更后刷新预览。
- 可以从 VS Code 停止本地 dev server。
- 优先使用项目中的 Clarify CLI，也支持通过配置指定自定义 CLI 路径。

## 命令

| 命令 | 作用 |
| --- | --- |
| `Clarify: Preview` | 为当前 Clarify 内容文件打开或更新预览。 |
| `Clarify: Refresh Preview` | 在内容、配置或依赖变更后重新加载预览面板。 |
| `Clarify: Stop Dev Server` | 停止本地 Clarify dev server，并结束预览会话。 |

## 设置

打开 VS Code 设置并搜索 `Clarify`，即可配置扩展。

| 设置 | 默认值 | 说明 |
| --- | --- | --- |
| `clarify.autoStartServer` | `true` | 打开项目时自动启动 Clarify dev server。 |
| `clarify.autoOpenPreview` | `true` | 在内容文件之间切换时自动更新预览。 |
| `clarify.openToSide` | `true` | 在当前编辑器旁边打开预览面板。 |
| `clarify.cliPath` | 空 | 使用指定的 `clarify` binary，而不是自动查找 CLI。 |

大多数项目不需要修改这些设置。只有在需要运行某个本地或自定义 Clarify CLI 构建时，才需要配置 `clarify.cliPath`。

## 支持的文件

扩展面向使用以下源文件的 Clarify 项目：

- `.md`
- `.mdx`
- `.openapi.json`
- `.openapi.yaml`
- `.openapi.yml`

扩展还会从本地 Clarify 服务读取项目信息，因此预览内容会与当前工作区使用的 CLI 版本保持一致。

## 常见问题

- **看不到预览命令**：确认当前编辑器中的文件位于 Clarify 内容根目录内，而不是任意文件。
- **预览打开了错误页面**：先保存文件，再运行 `Clarify: Refresh Preview`。
- **dev server 启动失败**：确认项目可以在终端中正常运行 `clarify dev`。
- **需要使用自定义 CLI**：将 `clarify.cliPath` 设置为目标 `clarify` binary 的绝对路径。
- **配置变更后预览仍然是旧内容**：先运行 `Clarify: Stop Dev Server`，再运行 `Clarify: Preview`。
