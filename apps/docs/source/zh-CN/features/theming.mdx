# 主题与样式

Clarify 当前提供的是稳定、收敛的主题配置：先选择内置主题预设，再用少量字段在预设基础上覆盖。

---

## 当前支持的配置

在 `clarify.ts` 中配置 `theme.preset`。当前只提供两个预设：`default` 和 `base`。

```ts
export default defineConfig({
  title: '我的文档',
  theme: {
    preset: 'base',
  },
})
```

`default` 采用 Clarify 自己的主题色系统，主色参考 SVG Logo 中的 `#00D492`，并搭配 `#00F6C9` 作为辅助色；`base` 没有颜色偏好，使用黑、白、灰构成标准主题。

剩余主题字段会在预设基础上覆盖。例如：

```ts
export default defineConfig({
  title: '我的文档',
  theme: {
    preset: 'base',
    tokens: {
      colors: {
        primary: '#00D492',
      },
    },
  },
})
```

`tokens.colors.primary` 接受 CSS 颜色字符串，用于覆盖当前预设的主色。配置值会进入运行时配置，供内置能力、插件或后续主题 API 使用；它不是完整样式覆盖能力。

---

## 不需要配置 Tailwind

Clarify 的渲染器内部使用 Tailwind CSS 4 和一组固定的组件样式。对使用者来说，这只是实现细节：

- 不需要创建 `tailwind.config.js`
- 不需要引入 Clarify 的内部样式入口
- 不应依赖内部 Tailwind 类名作为稳定 API

如果你的业务站点本身也使用 Tailwind，它与 Clarify 文档站点的内部构建管线是两件事。

---

## 暂不提供自定义 CSS 入口

当前版本没有稳定的“项目级自定义 CSS 文件”公共 API，也没有保证可覆盖的 `.clarify-*` 类名约定。

如果需要更深层的视觉定制，建议优先：

- 使用 `theme.preset` 选择接近目标风格的预设
- 使用 `theme.tokens.colors.primary` 覆盖品牌主色
- 使用 `logo` / `favicon` / `navbar` 等配置调整站点识别和入口
- 通过插件能力参与构建流程，而不是直接修改渲染器内部样式

---

## 深色模式

渲染器内置了亮色、暗色和系统偏好检测，并在导航栏提供主题切换。当前没有 `theme.mode` 配置项；用户选择会保存在浏览器本地存储中。
