# 国际化

Clarify 支持原生多语言文档。开启 `i18n` 后，内容按 locale 目录组织，路由、导航、语言切换器和缺失翻译回退会自动处理。

---

## 目录结构

启用国际化时，内容目录下每个 locale 对应一个子目录：

```txt
source/
├── zh-CN/
│   ├── index.mdx
│   ├── getting-started.mdx
│   └── guide/
│       └── configuration.mdx
└── en-US/
    ├── index.mdx
    ├── getting-started.mdx
    └── guide/
        └── configuration.mdx
```

如果默认语言是 `zh-CN`：

| 文件 | 路由 |
|------|------|
| `source/zh-CN/index.mdx` | `/` |
| `source/zh-CN/getting-started.mdx` | `/getting-started` |
| `source/en-US/index.mdx` | `/en-US` |
| `source/en-US/getting-started.mdx` | `/en-US/getting-started` |

---

## 配置示例

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  i18n: {
    defaultLocale: 'zh-CN',
    missing: 'fallback',
    locales: [
      { code: 'zh-CN', label: '简体中文', dir: 'ltr' },
      { code: 'en-US', label: 'English', dir: 'ltr' },
    ],
  },
})
```

---

## 配置字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `defaultLocale` | `string` | `locales[0].code` | 默认语言；默认语言不加 URL 前缀 |
| `missing` | `'fallback' \| '404' \| 'hide'` | `'fallback'` | 缺失翻译处理策略 |
| `locales` | `Array<{ code, label, dir? }>` | — | 可用语言列表 |

`locales` 中每一项：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `string` | locale 代码，例如 `zh-CN`、`en-US` |
| `label` | `string` | 在语言切换器中展示的名称 |
| `dir` | `'ltr' \| 'rtl'` | 文档方向，可选 |

---

## 缺失翻译策略

### `fallback`

如果某个语言缺少页面，使用默认语言内容生成对应 locale 路由。

适合产品文档渐进翻译：先保证所有语言路径可访问，再逐步补齐翻译。

### `404`

缺失页面不生成对应语言路由，用户访问时由 404 页面处理。

适合严格区分已翻译和未翻译内容的站点。

### `hide`

缺失页面不生成对应语言路由。使用 `pages: 'FileTree'` 自动导航时，未翻译页面也不会出现在该语言的导航中。

如果使用手动 `pages` 配置，需要确保导航项只引用目标语言存在的页面，或改用 `fallback` 策略。

---

## 本地化文本

配置中的可见文本支持字符串或按 locale 分组的对象：

```ts
export default defineConfig({
  navbar: {
    links: [
      {
        label: {
          'zh-CN': '文档',
          'en-US': 'Docs',
        },
        href: '/getting-started',
      },
    ],
  },
})
```

支持本地化的配置字段包括：

- `navbar.links[].label`
- `banner.content`
- `footer.copyright`
- `pages[].group`
- `pages[].pages[].title`

---

## 语言切换

开启多个 locale 后，Clarify 会在顶部导航显示语言切换器。

切换时会优先使用当前页面的同源翻译路径。如果目标语言没有对应页面，并且没有生成 fallback 路由，则不会提供该语言的跳转项。

---

## 最佳实践

- 保持不同语言目录下的相对路径一致。
- 默认语言内容先完整上线，再逐步补齐其他语言。
- `pages` 中引用不带 locale 的基础路径，例如 `getting-started`，不要写 `zh-CN/getting-started`。
- 对产品导航、分组名和页面标题使用本地化对象，避免不同语言混用。
