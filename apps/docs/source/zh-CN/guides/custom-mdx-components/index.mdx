# 自定义 MDX 组件

Clarify 支持在 `.mdx` 中直接导入项目里的 React 组件。一次性组件可以和文章放在同一个目录里，导入路径短，也能清楚表达它只属于这篇内容。

```text
custom-mdx-components/
├── index.mdx
└── SnakeGame.tsx
```

```mdx
import { SnakeGame } from './SnakeGame'
```

## 放在哪里

<Steps>
  <Step title="文章专用组件">
    放在文章目录旁边，例如 `guide/index.mdx` 与 `guide/Diagram.tsx`。这是默认推荐方式。
  </Step>
  <Step title="栏目共享组件">
    放在该栏目的 `_components` 目录，由同一栏目内的几篇文章复用。
  </Step>
  <Step title="全站公共组件">
    只有真正跨栏目、跨语言复用时，才提升到 `source/components`。
  </Step>
</Steps>

<Callout type="tip">
  Clarify 只把 `.md` 和 `.mdx` 扫描为内容路由，同目录的 `.tsx` 文件不会生成额外页面。
</Callout>
