# 什么是 Clarify？

Clarify 是一款面向现代开发者文档的 **开源文档发布工具**。它适合那些既希望文档体验足够精致，又不想放弃源码所有权、Git 工作流和自托管能力的团队。

它把 MDX、OpenAPI、类型化 TypeScript 配置、原生多语言、主题定制、多设备响应式布局，以及 AI 可读输出整合到一个 local-first 的发布流程中。文档仍然留在仓库里，站点可以构建为静态 HTML 自托管，读者则能在桌面、平板和手机上获得一致的现代阅读体验。

## 为什么需要 Clarify

很多团队想要接近优秀托管文档平台的体验，但又希望保持开源项目的控制权、可移植性和长期可维护性。

Clarify 想填补的正是这个空白：

- **默认开源** — 文档、配置、OpenAPI 规范和主题设置都可以放在仓库中。
- **源码所有权** — 内容可以和产品代码一起 Review、版本化和发布。
- **静态、自托管** — 生产产物可以部署到任意静态托管平台。
- **现代阅读体验** — 导航、搜索、代码块、API 面板、暗/亮主题和响应式布局都内置可用。

---

## 把 OpenAPI 当作一等内容层

Clarify 不只是把 OpenAPI 渲染成一个独立的 API Reference。

你可以生成完整的接口参考页面，也可以在任意 MDX 文档中直接引用某一个 API operation。这样一篇指南可以先解释业务流程，再在读者需要的步骤旁边放入对应接口、参数、示例和请求代码。

这让 API 文档不再被隔离在单独区域，而是能自然融入教程、上手流程、SDK 指南和产品说明中。

## 原生多语言文档

Clarify 内置国际化支持，适合把多语言文档放在同一个站点结构下维护，同时保持每种语言的内容独立。

团队可以在 `clarify.ts` 中配置 locale，把英文和翻译页面并排维护，并让 Clarify 处理本地化路由、导航文案和 fallback 行为。

## 暗/亮主题与品牌控制

Clarify 内置自动暗色 / 亮色主题支持，也提供灵活的主题配置和 theme token。

这意味着团队可以在不重写渲染器的前提下，让文档站点贴合自己的品牌。开源项目可以拥有更精致的文档体验，产品团队也可以让文档和自身视觉系统保持一致。

## 适配多尺寸设备

Clarify 从一开始就面向桌面、平板和手机阅读体验设计。

导航、内容宽度、代码块、侧边栏、API 面板和页面操作都会根据屏幕尺寸调整。无论用户是在电脑上阅读完整指南，在平板上查看 API，还是在手机上快速查一个步骤，文档都能保持可用。

---

## 面向 AI 的文档输出

现代文档不仅被人阅读，也会被 AI Agent 和开发工具读取。

Clarify 会生成原始 Markdown 内容、OpenAPI 产物和 `llms.txt`，让文档可以被助手、搜索索引、内部知识库和自动化工具消费。页面也提供复制原始内容、打开原始链接等操作，让同一份源码同时服务人和机器。

## 谁适合使用 Clarify？

Clarify 特别适合：

- **开源项目** — 想要更好的文档体验，但不希望把内容迁出 Git。
- **API 平台团队** — 需要把 OpenAPI Reference 和任务型指南放在同一个站点中。
- **开发者工具团队** — 重视自托管、可移植性和长期内容所有权。
- **技术写作 / DevRel 团队** — 希望使用 MDX、组件、多语言和主题配置，但不想维护一套自研文档系统。

---

## 一句话总结

Clarify 是一个开源、源码自有的文档发布层，覆盖 MDX、OpenAPI、多语言文档、响应式主题和 AI-ready 知识库。
