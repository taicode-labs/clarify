# Guides

Clarify's user guides converge around real tasks into four core documents, avoiding having the same capability scattered across multiple pages.

---

## Recommended reading order

1. [Getting Started](/getting-started) — install the CLI, create pages, and build a static site.
2. [Writing Content](/guides/writing-content) — understand MDX, frontmatter, file routing, and built-in components.
3. [Configuring the Site](/guides/navigation) — configure navigation, theme, multiple languages, and the deployment subpath.
4. [API Documentation](/features/openapi) — generate a reference with OpenAPI and embed endpoints in your guides.
5. [Publishing](/guides/deployment) — deploy static artifacts and output `llms.txt` for AI tools to read.

---

## Migration scenarios

- [Migrate from Mintlify](/guides/migrate-from-mintlify) — smoothly migrate existing docs to Clarify.
