# Migrate from Mintlify to Clarify

Clarify's design draws on Mintlify's best practices and offers a compatible configuration format to make migration as smooth as possible.

---

## Configuration compatibility

Mintlify's `docs.json` and Clarify's `clarify.json` fields are largely compatible:

| Mintlify | Clarify | Notes |
|----------|---------|------|
| `name` | `title` | Synonymous |
| `description` | `description` | Direct mapping |
| `logo` | `logo` | Direct mapping |
| `colors.primary` | `theme.tokens.colors.primary` | Path adjusted, can be used together with `theme.preset` |

Clarify accepts unknown Mintlify fields and silently ignores them (with a warning), providing a smooth migration path.

---

## Migration steps

<Steps>
  <Step title="Rename the configuration file">

    ```bash
    mv docs.json clarify.json
    ```

  </Step>
  <Step title="Adjust fields">

    ```json
    // Before (docs.json)
    {
      "name": "My Docs",
      "colors": {
        "primary": "#00D492"
      }
    }

    // After (clarify.json)
    {
      "title": "My Docs",
      "theme": {
        "preset": "default",
        "tokens": {
          "colors": {
            "primary": "#00D492"
          }
        }
      }
    }
    ```

  </Step>
  <Step title="Migrate content files">

    Mintlify uses `.mdx` or `.md` files, and Clarify supports MDX as well:

    - Copy content files into the `source/` directory
    - Make sure the file structure matches the routing expectations
    - Check and update the frontmatter format

  </Step>
  <Step title="Install dependencies and build">

    ```bash
    pnpm add -D @clarify-labs/cli
    pnpm exec clarify build
    ```

  </Step>
</Steps>

## Differences

| Feature | Mintlify | Clarify |
|------|----------|---------|
| Hosting | Hosted service | Self-hosted, built as static files |
| Route configuration | `navigation` field | Explicit `tabs` config + file routing |
| Component library | Built-in Mintlify components | MDX + Clarify built-in components + custom React components |
| OpenAPI | Hosted platform capability | OpenAPI files as a first-class content source, can generate a reference and embed endpoints |
| i18n | Supported | Built-in locale directories, language switching, and a missing-translation strategy |
| AI reading | Platform capability | Static output of Markdown, OpenAPI, and `llms.txt` |
