# Configure Site

Site behavior is configured in `clarify.ts`, `clarify.js`, or `clarify.json`. `clarify.ts` is recommended because it provides type hints.

---

## Basic config

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  title: 'My Docs',
  description: 'Documentation built with Clarify',
  navbar: {
    links: [
      { label: 'Docs', href: '/getting-started' },
      { label: 'GitHub', href: 'https://github.com/example/project', external: true },
    ],
  },
})
```

---

## Navigation tabs

Use `tabs` to define product-level sections and sidebar entries:

```ts
export default defineConfig({
  tabs: [
    {
      tab: { 'zh-CN': '文档', 'en-US': 'Docs' },
      icon: 'BookOpen',
      pages: [
        {
          group: { 'zh-CN': '核心路径', 'en-US': 'Core Path' },
          pages: [
            { page: 'getting-started', title: { 'zh-CN': '快速开始', 'en-US': 'Get Started' } },
            { openapi: 'api.openapi.json', title: 'API Reference' },
          ],
        },
      ],
    },
  ],
})
```

`page` points to an MDX route. `openapi` points to an OpenAPI spec file and creates a generated API Reference page.

## Localization

```ts
export default defineConfig({
  i18n: {
    defaultLocale: 'zh-CN',
    missing: 'fallback',
    locales: [
      { code: 'zh-CN', label: '简体中文', dir: 'ltr' },
      { code: 'en-US', label: 'English', dir: 'ltr' },
    ],
  },
})
```

With `missing: 'fallback'`, missing localized pages fall back to the default locale. Add English files under `source/en-US/` for pages that should be fully localized.
