# Writing Docs

Clarify uses the `source/` directory as the content root. Write Markdown or MDX files there, and Clarify turns them into routes automatically.

---

## File routes

| File | Route |
|------|-------|
| `source/index.mdx` | `/` |
| `source/getting-started.mdx` | `/getting-started` |
| `source/guides/index.mdx` | `/guides` |
| `source/guides/writing-content.mdx` | `/guides/writing-content` |

For localized sites, place files under locale folders:

```txt
source/
├── zh-CN/
│   └── index.mdx
└── en-US/
    └── index.mdx
```

---

## MDX basics

Use regular Markdown for prose and MDX for components:

```mdx
# Guide title

Use Markdown for the main content.

<MyComponent name="Clarify" />
```

Clarify also supports code blocks, tables, frontmatter, and renderer-provided documentation components.

---

## Built-in MDX components

Clarify provides default MDX components that are available automatically in `.mdx` pages.
You do not need to import them before use.

- `Callout`
- `CardGroup` / `Card`
- `Button`
- `CodeGroup`
- `OpenApiOperation`
- `OpenApiDocument`

Example:

```mdx
<Callout type="tip">
  Use built-in components for structured content and callouts.
</Callout>

<CardGroup cols={2}>
  <Card title="Quick start" icon="rocket" href="/getting-started">
    Install the CLI and create a Clarify docs site.
  </Card>
  <Card title="API docs" icon="braces" href="/features/openapi">
    Embed OpenAPI content directly in MDX.
  </Card>
</CardGroup>
```

For project-specific components, import them explicitly at the top of the MDX file:

```mdx
import { Alert } from '../components/Alert'
```

## Recommended structure

```txt
source/
├── index.mdx
├── getting-started.mdx
├── guides/
│   ├── index.mdx
│   └── writing-content.mdx
├── reference/
│   └── clarify-config.mdx
└── api.openapi.json
```

Keep task-oriented guides in `guides/`, stable API or configuration details in `reference/`, and OpenAPI specs beside the documentation they support.
