# Deployment

This page explains Clarify's publishing capabilities: build-time SSR, static generation, common hosting targets, route prefixes, raw content output, and `llms.txt`.

Clarify supports SSR rendering, but it uses **build-time SSR**: `clarify build` renders every route into HTML during the build. The production artifact is still a pure static site, so no Node.js server runtime is required after deployment.

---

## How SSR works

Clarify's production rendering model is SSG (Static Site Generation) plus hydration:

1. The build environment runs `pnpm exec clarify build`.
2. The CLI uses a Vite SSR bundle to render every documentation route in Node.js.
3. Each route is written as its own `index.html`, alongside JS/CSS assets, raw Markdown/OpenAPI artifacts, and `llms.txt`.
4. Visitors receive pre-rendered HTML first. After JavaScript loads, the page hydrates into a React app for client-side navigation.

This means:

- ✅ first-load HTML, SEO-friendly output, and direct deep links are supported;
- ✅ deployment only needs static file hosting;
- ✅ Vercel, Netlify, GitHub Pages, object storage, CDNs, and Nginx all work;
- ❌ production request-time SSR servers are not required or provided.

## Build output

Run:

```bash
pnpm exec clarify build
```

The default output directory is `output/`. You can override it:

```bash
pnpm exec clarify build --output dist
```

A typical build looks like this:

```txt
output/
├── index.html
├── getting-started/
│   └── index.html
├── assets/
├── llms.txt
├── index.md
└── api.openapi.json
```

Every route has its own HTML file for fast first loads, direct linking, and search-friendly output. After JavaScript loads, users get client-side navigation. Deploy only the `output/` directory; do not deploy temporary SSR build files.

## Hosting targets

### Vercel

1. Push the repository to GitHub, GitLab, or Bitbucket.
2. Import the project in Vercel.
3. Build command: `pnpm exec clarify build`.
4. Output directory: `output`.

### Netlify

1. Connect the Git repository.
2. Build command: `pnpm exec clarify build`.
3. Publish directory: `output`.

### GitHub Pages

If the repository is named `my-repo` and the site URL is `https://username.github.io/my-repo/`, configure:

```ts
export default defineConfig({
  routePrefix: '/my-repo/',
})
```

Example GitHub Actions workflow:

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 9
      - run: pnpm install
      - run: pnpm exec clarify build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./output
```

### CDN / Nginx

Upload `output/` to your web root:

```bash
rsync -avz output/ user@server:/var/www/html/
```

Nginx example:

```nginx
server {
    listen 80;
    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## Raw content output

Clarify exposes raw content during development and production builds. This is useful for search, knowledge bases, and AI tools.

| Page route | Raw content URL |
|------------|-----------------|
| `/` | `/index.md` |
| `/getting-started` | `/getting-started.md` |
| `/features/openapi` | `/features/openapi.md` |
| `/api` | `/api.openapi.json` |

The page content actions use these URLs to copy raw page content, copy raw content links, or let AI tools reference stable Markdown/OpenAPI addresses.

## `llms.txt`

The build output includes `llms.txt`, listing the site title, description, MDX pages, and OpenAPI entry points:

```txt
# Clarify

> Open-source documentation publishing for MDX and OpenAPI.

## Docs
- [Get Started](/getting-started.md)
- [API Documentation](/features/openapi.md)

## OpenAPI
- [API Reference](/api.openapi.json)
```

It is useful for:

- AI coding assistants;
- internal knowledge-base indexing;
- documentation quality tools;
- search or Q&A systems as a discovery manifest.

If `routePrefix` is configured, links in `llms.txt` automatically include the same prefix.

## Pre-publish checklist

- The build environment satisfies Node.js `>=20.0.0` and pnpm `>=9.0.0`.
- `pnpm exec clarify build` completes successfully.
- `output/` contains the expected HTML, assets, raw Markdown/OpenAPI files, and `llms.txt`.
- `routePrefix` is configured when deploying under a subpath.
- Raw OpenAPI output does not include internal-only information.
- The hosting platform points to `output` or your custom output directory.
- Production can be deployed as a static site; no Node.js SSR service needs to be started.
