# Product Roadmap

Clarify is evolving toward a focused documentation publishing workflow for MDX and OpenAPI.

---

## Current priorities

| Area | Direction |
|------|-----------|
| Authoring | Improve MDX authoring ergonomics and built-in docs components. |
| OpenAPI | Make API Reference pages more complete and easier to embed in guides. |
| Static output | Keep builds fast, deterministic, and friendly to SEO and AI tools. |
| i18n | Make localized docs predictable with explicit fallback behavior. |
| Extensibility | Stabilize plugin APIs for teams that need custom pipelines. |

---

## Principles

- Keep docs content close to source code.
- Prefer static output that can be hosted anywhere.
- Treat OpenAPI specs as first-class content.
- Keep configuration explicit and readable.
