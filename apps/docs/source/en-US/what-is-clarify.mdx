# What is Clarify?

Clarify is an **open-source documentation publishing tool** for teams that want polished developer documentation without giving up source ownership.

It brings MDX, OpenAPI, typed TypeScript configuration, native internationalization, theme customization, responsive layouts, and AI-readable output into one local-first publishing workflow. Your docs stay in Git, your site can be self-hosted as static HTML, and your readers get a modern experience across desktop, tablet, and mobile.

---

## Why Clarify exists

Many teams want documentation that feels as polished as the best hosted platforms, but they also want the control and portability of an open-source project.

Clarify is designed to fill that gap:

- **Open-source by default** — docs, configuration, OpenAPI specs, and theme settings live in your repository.
- **Source ownership** — content is reviewed, versioned, and shipped together with the product code.
- **Static and self-hostable** — production output can be deployed to any static hosting platform.
- **Modern reader experience** — navigation, search, code blocks, API panels, dark/light themes, and responsive layouts are built in.

---

## OpenAPI as a first-class content layer

Clarify treats OpenAPI as more than a separate API reference.

You can publish full API reference pages, but you can also reference individual API operations directly inside any MDX document. That means a guide can explain a workflow and place the exact endpoint, parameters, examples, and request code beside the step where readers need it.

This makes API documentation feel integrated with tutorials, onboarding flows, SDK guides, and product documentation instead of being isolated in a separate section.

---

## Native multilingual documentation

Clarify has native internationalization support, so multilingual documentation can share the same site structure while keeping localized content separate.

Teams can configure locales in `clarify.ts`, keep English and translated pages side by side, and let Clarify handle locale-aware routes, navigation labels, and fallback behavior.

---

## Themes, dark mode, and brand control

Clarify includes automatic dark and light theme support, flexible theme configuration, and theme tokens that can be controlled from the project configuration.

This lets teams adapt the documentation site to their brand without rebuilding the renderer from scratch. Open-source projects can ship docs that feel polished, while product teams can align the site with their visual system.

---

## Responsive across device sizes

Clarify is built for desktop, tablet, and mobile reading from the start.

Navigation, content width, code blocks, sidebars, API panels, and page actions adapt to different screen sizes, so documentation remains usable whether readers are browsing on a laptop, reviewing an API on a tablet, or checking a guide from a phone.

---

## AI-ready by design

Modern documentation is read by people and by AI agents.

Clarify generates raw Markdown content, OpenAPI artifacts, and `llms.txt` so documentation can be consumed by developer tools, assistants, search pipelines, and internal knowledge systems. Pages also expose actions for copying raw content and opening raw links, making the same source useful for both humans and machines.

---

## Who should use Clarify?

Clarify is especially useful for:

- **Open-source projects** that want great documentation without moving content out of Git.
- **API platform teams** that need OpenAPI references and task-oriented guides in one place.
- **Developer tooling teams** that care about self-hosting, portability, and long-term content ownership.
- **Technical writing and DevRel teams** that want MDX, components, localization, and theme control without maintaining a custom docs stack.

---

## In one sentence

Clarify is an open-source, source-owned documentation publishing layer for MDX, OpenAPI, multilingual docs, responsive themes, and AI-ready knowledge bases.
