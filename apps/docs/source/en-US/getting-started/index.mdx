# Getting Started

This page explains how Clarify works through the shortest possible path: install the CLI, create content, start the local site, and build static artifacts.

Clarify = MDX content + `clarify.ts` configuration + OpenAPI specs + static site generation. You mainly maintain `source/` and the configuration file, while Clarify handles the dev server, routing, rendering, and builds.

---

## 1. Prepare your environment

You need:

- Node.js ‚â• 20
- pnpm ‚â• 9

Create a project and install the CLI:

```bash
mkdir my-docs
cd my-docs
pnpm init
pnpm add -D @clarify-labs/cli
```

Add scripts to `package.json`:

```json
{
  "scripts": {
    "dev": "clarify dev",
    "build": "clarify build --output output"
  }
}
```

---

## 2. Add site configuration

We recommend using `clarify.ts`, which gives you type hints and makes it easier to add plugins later.

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  title: 'My Docs',
  description: 'A documentation site built with Clarify',
  navbar: {
    links: [
      { label: 'Docs', href: '/getting-started' },
      { label: 'GitHub', href: 'https://github.com/example/project', external: true },
    ],
  },
})
```

Simple projects can also use `clarify.json`:

```json
{
  "title": "My Docs",
  "description": "A documentation site built with Clarify"
}
```

See the [Configuration Reference](/reference/clarify-config) for all fields.

| File | Route |

## 4. Start the dev server

```bash
pnpm dev
```

Open the local address printed in the terminal. After you edit MDX, configuration, or OpenAPI files, the dev server regenerates the pages.

---

## 5. Build the production version

```bash
pnpm build
```
