# Features

Clarify has a clear positioning: it brings "writing docs, generating an API reference, publishing a static site, and letting AI read the docs" into the same local-first workflow.

## The user path

1. **Start the site**: install the CLI and create `clarify.ts` and `source/`.
2. **Write content**: use MDX to organize the home page, guides, reference pages, and component examples, and use clear H2/H3 headings to improve the table of contents and search.
3. **Connect the API**: drop in an OpenAPI spec, generate the reference, embed key endpoints in your guides, and provide copyable request code for developers.
4. **Configure the information architecture**: use tabs, groups, navigation, redirects, the footer, and multi-language config to organize reading paths.
5. **Publish and serve to AI/search**: build the static site while also outputting raw content files, `llms.txt`, and the Pagefind full-text index.
6. **Extend team workflows**: add plugins when you need content governance, translation checks, or custom artifacts.

---

## Design philosophy

1. **Start from user tasks**: docs are organized around "start, write, configure, API, publish, extend" rather than dedicating one page to each internal capability.
2. **Content close to code**: MDX, OpenAPI, and configuration all live in the repository for easy version management and review.
3. **Static-first**: build artifacts can be deployed to any static hosting platform and can also be read by search engines and AI tools.
4. **Progressive extension**: the default capabilities work out of the box; add plugins only when you need search, governance, or extra output.
