# Changelog

---
## v0.10.2

### Added

- Added a dedicated Markdown content pipeline: `.md` uses Markdown/HTML semantics while `.mdx` keeps strict JSX semantics
- Documented when to choose `.md` or `.mdx`, including their syntax differences

### Improved

- Replaced custom heading ID and syntax-highlighting implementations with the community-maintained `rehype-slug` and `@shikijs/rehype` plugins
- Improved code-block language detection, metadata handling, Mermaid rendering, and unknown-language fallback

---

## v0.8.2

### Improved

- Bumped the Clarify workspace package versions to `0.8.2`
- Added support for project variables so project-wide fixed content can be defined and reused centrally
- Regenerated the CLI bundles so `clarify --version` matches the released patch version
- Rebuilt the documentation site output so the roadmap and changelog pages reflect the latest patch release

---

## v0.8.0

### Added

- Added runtime UI slots for plugins, including SSR-aware lazy loading and support for replacing selected built-in surfaces
- Added the `build:assets` plugin hook so plugins can collect and emit static assets during production builds
- Added locale-specific OpenAPI virtual modules so multilingual sites can render and embed the correct localized API specification
- Added Vitest coverage across CLI commands, configuration loading, routing, SSG, OpenAPI, content artifacts, SEO artifacts, source links, and renderer utilities

### Improved

- Improved OpenAPI rendering with per-page spec data injection, cleaner virtual module generation, and better SSR behavior
- Improved prose table rendering with wrappers, nowrap headers, and safer inline-code layout inside table cells
- Improved plugin type exports and runtime slot naming for clearer integration boundaries
