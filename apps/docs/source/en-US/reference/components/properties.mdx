# Properties / Property

`Properties` and `Property` describe configuration, component props, and data fields. They handle long descriptions better than wide tables and remain readable on narrow screens.

## Usage

```mdx
<Properties>
  <Property name="routePrefix" type="string">
    Deployment subpath, such as `/docs/`.
  </Property>
  <Property name="tabs" type="ClarifyTabsConfig">
    Configures top-level categories and sidebar navigation.
  </Property>
</Properties>
```

<Properties>
  <Property name="routePrefix" type="string">Deployment subpath, such as `/docs/`.</Property>
  <Property name="tabs" type="ClarifyTabsConfig">Configures top-level categories and sidebar navigation.</Property>
</Properties>

## Props

### Properties

<Properties>
  <Property name="children" type="ReactNode">One or more `Property` elements.</Property>
  <Property name="className" type="string">Optional class name for the outer container.</Property>
  <Property name="spacingClassName" type="string">Optional spacing class name. Defaults to `my-6`.</Property>
</Properties>

### Property

<Properties>
  <Property name="name" type="string">Field or property name.</Property>
  <Property name="type" type="string">Optional type label.</Property>
  <Property name="children" type="ReactNode">Detailed description with MDX support.</Property>
</Properties>

<Callout type="tip">Use a table for short, comparable data. Use `Properties` when descriptions contain paragraphs, links, or examples.</Callout>
