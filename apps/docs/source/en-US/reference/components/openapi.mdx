# OpenAPI Components

`OpenApiOperation` embeds one operation in a task-oriented guide. `OpenApiDocument` embeds a complete or tag-filtered API reference. Both render authentication, parameters, request bodies, responses, and request examples.

## One operation

```mdx
<OpenApiOperation specPath="/api" operationId="demoBasicRequest" />
```

`operationId` must uniquely identify an operation in the specification.

Rendered result:

<OpenApiOperation specPath="/api" operationId="demoBasicRequest" />

The play button in the endpoint bar opens an interactive request tester. Users can select a server and authentication scheme, edit parameters and the request body, then inspect the status, duration, headers, and response body.

## Inline request tester

Embed the same request tester directly in a guide when trying the endpoint is part of the workflow:

```mdx
<OpenApiRequest specPath="/api" operationId="demoBasicRequest" requestExample="published" />
```

`requestExample` selects the initial OpenAPI request body or parameter example by key. Matching parameter and request body examples are applied together.

<OpenApiRequest specPath="/api" operationId="demoBasicRequest" requestExample="published" />

## Complete reference

```mdx
<OpenApiDocument specPath="/api" />
```

Limit the reference by tags:

```mdx
<OpenApiDocument specPath="/api" tagFilter={["Users", "Teams"]} />
```

## Props

| Component | Prop | Type | Description |
|-----------|------|------|-------------|
| `OpenApiOperation` | `specPath` | `string` | Configured OpenAPI specification path |
| `OpenApiOperation` | `operationId` | `string` | Unique OpenAPI operation identifier |
| `OpenApiRequest` | `specPath` | `string` | Configured OpenAPI specification path |
| `OpenApiRequest` | `operationId` | `string` | Unique OpenAPI operation identifier |
| `OpenApiRequest` | `requestExample` | `string` | Initial request body or parameter example key |
| `OpenApiDocument` | `specPath` | `string` | Configured OpenAPI specification path |
| `OpenApiDocument` | `tagFilter` | `string[]` | Includes only operations with matching tags |

<Callout type="tip">Let guides explain tasks and decisions while OpenAPI components own interface facts, preventing hand-written parameter tables from drifting.</Callout>

See [OpenAPI documentation](/features/openapi) and [Embed OpenAPI in MDX](/openapi/embedding) for configuration details.
