# Callout

`Callout` highlights information that affects understanding, decisions, or task outcomes. It supports semantic variants, an optional title, and an icon.

## Variants

<Callout type="info" title="Info">Provide context needed for the current task.</Callout>
<Callout type="note" title="Note">Add detail that the main text does not expand.</Callout>
<Callout type="tip" title="Tip">Recommend a faster or safer approach.</Callout>
<Callout type="warning" title="Warning">Flag potential failures, permissions, or compatibility issues.</Callout>
<Callout type="danger" title="Danger">Mark destructive or security-sensitive actions.</Callout>
<Callout type="success" title="Success">Confirm completion or successful validation.</Callout>

```mdx
<Callout type="warning" title="Production">
  Update the reverse proxy rules after changing `routePrefix`.
</Callout>
```

## Props

<Properties>
  <Property name="type" type="'info' | 'note' | 'tip' | 'warning' | 'danger' | 'success'">Semantic color and default icon. Defaults to `'info'`.</Property>
  <Property name="title" type="ReactNode">Heading derived from `type` by default; pass an empty value to hide it.</Property>
  <Property name="icon" type="boolean">Whether to show the variant icon. Defaults to `true`.</Property>
  <Property name="children" type="ReactNode">Required body with MDX support.</Property>
  <Property name="className" type="string">Optional class name for the container.</Property>
</Properties>

## Guidelines

- Keep one conclusion per callout and use a short phrase for its title.
- Use `Note` for ordinary background so the page is not dominated by colored panels.
- Reserve `danger` for irreversible or security-sensitive risks; use `warning` for ordinary cautions.
