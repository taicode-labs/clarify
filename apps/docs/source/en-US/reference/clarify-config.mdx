# Configuration Reference

Clarify project config lives in `clarify.ts`, `clarify.js`, or `clarify.json`. Loading priority is:

```txt
clarify.ts > clarify.js > clarify.json
```

Use `clarify.ts` when you want type hints or plugins. Use `clarify.json` for simple static configuration.

---

## Example

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  title: 'Clarify Docs',
  description: 'Open-source documentation publishing for MDX and OpenAPI',
  logo: '/logo.svg',
  favicon: '/favicon.ico',
  theme: {
    preset: 'default',
    editor: true,
    tokens: {
      colors: {
        primary: { light: '#00D492', dark: '#34D399' },
        accent: '#0D9488',
        muted: { light: '#64748b', dark: '#a1a1aa' },
      },
    },
  },
  navbar: {
    links: [
      { label: { 'zh-CN': '文档', 'en-US': 'Docs' }, href: '/getting-started' },
      { label: 'GitHub', href: 'https://github.com/taicode-labs/clarify', external: true },
    ],
  },
  i18n: {
    defaultLocale: 'zh-CN',
    missing: 'fallback',
    locales: [
      { code: 'zh-CN', label: '简体中文', dir: 'ltr' },
      { code: 'en-US', label: 'English', dir: 'ltr' },
    ],
  },
  tabs: [
    {
      tab: { 'zh-CN': '文档', 'en-US': 'Docs' },
      icon: 'BookOpen',
      pages: 'FileTree',
    },
  ],
})
```

## Top-level fields

| Field | Description |
|-------|-------------|
| `title` | Site title used by the shell and generated metadata. |
| `description` | Site description used for SEO and `llms.txt`. |
| `logo` | Logo path or light/dark logo variants. |
| `favicon` | Favicon path or light/dark variants. |
| `theme` | Theme preset, token overrides, and `theme.editor` for exposing the live theme editor in built sites. Dev mode enables the editor automatically. |
| `routePrefix` | Deployment subpath, such as `/docs/`. |
| `navbar` | Top navigation links. |
| `footer` | Footer links, socials, and copyright. |
| `i18n` | Locale configuration and missing-content behavior. |
| `tabs` | High-level navigation sections and sidebar structure. |
| `ssg` | Static generation behavior. |
| `plugins` | Build-time plugins, available in JS/TS config files. |

## `theme`

```ts
theme: {
  preset?: 'default' | 'base'
  tokens?: ClarifyThemeTokensConfig
  layout?: ClarifyThemeLayoutConfig
  editor?: boolean
}
```

Theme tokens are partial overrides of the selected preset. Color tokens can be plain strings or mode-aware values:

```ts
export default defineConfig({
  theme: {
    preset: 'base',
    editor: true,
    tokens: {
      colors: {
        primary: { light: '#00D492', dark: '#34D399' },
        accent: '#0D9488',
        muted: { light: '#64748b', dark: '#a1a1aa' },
      },
    },
  },
})
```

`theme.editor` exposes the live theme editor in built sites. Dev mode enables the editor automatically.

## Custom 404 page

Create `source/404.mdx` to customize the not-found page:

```mdx
# Page not found

The page you are looking for does not exist or has moved.

- [Get started](/getting-started)
- [Configuration reference](/reference/clarify-config)
```

Clarify serves it at `/404`, uses it for unmatched client routes, and writes `output/404.html` during static generation. File-tree navigation hides `404` automatically.

## Localized text

Many config fields accept localized text:

```ts
type ClarifyLocalizedText = string | Record<string, string>
```

Example:

```ts
{
  label: {
    'zh-CN': '文档',
    'en-US': 'Docs',
  }
}
```

## OpenAPI page filters

OpenAPI entries in `tabs[].pages` can publish either the full spec or a filtered subset of operations.

```ts
{
  openapi: 'api.openapi.json',
  title: 'Projects API',
  filter: {
    tags: ['Projects'],
  },
}
```

`filter.tags` matches OpenAPI `operation.tags`. If any operation tag matches one of the configured tags, the operation appears on that page. Without `filter`, the page renders every operation in the spec.

Filtered pages get a tag-based route suffix. For example, `api.openapi.json` with `filter.tags: ['Projects']` is served at `/api/projects`.
