## Automatic registration

These components are automatically available only in Clarify MDX pages. Import project-specific React components explicitly at the top of the page:

```mdx
import { Status } from '../../components/Status'

<Status value="ready" />
```

<Callout type="tip">
  Leave blank lines between block Markdown, fenced code, and JSX components so each block is parsed correctly.
</Callout>

## Composition guidelines

- Use `Steps` for sequence and `Tabs` for mutually exclusive choices, not as substitutes for ordinary sections.
- Use `Callout` for decision-relevant information and `Note` for context that does not change the main task.
- Use `Card` to navigate to another page and `Button` for a clear next action.
- Use `CodeGroup` to compare equivalent code and `Collapse` for optional detail.
- Use `AccordionGroup` for related disclosures, `FileTree` for directory structures, and `Figure` for captioned images.
- Use `Tooltip` only for short supplementary definitions; keep required information visible in the page.
- Use `Mermaid` for diagrams whose source should remain versionable and searchable.
- Pass data-driven Markdown strings to `Markdown`; content written directly in `.mdx` needs no wrapper.
