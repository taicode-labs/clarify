# Built-in Components

Clarify registers a set of documentation components automatically in MDX.
You do not need to import them manually in `.mdx` content.

---

## Built-in MDX components

The following components are available directly in MDX:

- `Callout`
- `CardGroup` / `Card`
- `Button`
- `CodeGroup`
- `OpenApiEndpoint`
- `OpenApiPage`

---

## OpenAPI components

### `OpenApiEndpoint`

Embed a single operation from an OpenAPI spec:

```mdx
<OpenApiEndpoint specPath="/api" path="/examples/basic-request" method="get" />
```

| Prop | Description |
|------|-------------|
| `specPath` | Route of the OpenAPI reference, such as `/api`. |
| `path` | Exact OpenAPI path key, such as `/examples/path-parameters/{pageId}`. |
| `method` | HTTP method, usually lowercase. |

### `OpenApiPage`

Embed a complete OpenAPI Reference page in MDX:

```mdx
<OpenApiPage specPath="/api" />
```

---

## Custom MDX components

Applications can provide additional MDX components through the renderer integration. These custom components must be imported explicitly in the MDX file:

```mdx
import { Alert } from '../components/Alert'

<Alert type="warning">
  This is a project-specific component.
</Alert>
```

Keep the renderer's public components stable and document their props for authors.
