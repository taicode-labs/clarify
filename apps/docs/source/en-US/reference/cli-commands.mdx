# CLI Reference

The Clarify CLI provides commands for local development, production builds, and project setup.

---

## Installation

Install the CLI globally and run commands directly:

```bash
npm install -g @clarify-labs/cli
```

Then use:

```bash
clarify init
clarify dev
clarify check
clarify build
```

If you prefer a project-scoped install, you can still add the CLI as a dev dependency.

---

## `clarify dev`

Start the local development server.

```bash
pnpm exec clarify dev
```

Common options include selecting the project root, content directory, output directory, and development port.

---

## `clarify build`

Build a production static site.

```bash
pnpm exec clarify build --output output
```

The build validates content and OpenAPI specs, renders pages, writes static assets, copies public files, and emits `llms.txt`. When `siteUrl` is configured, it also writes `sitemap.xml` and `robots.txt`.

---

## `clarify check`

Validate the documentation project without producing a build.

```bash
pnpm exec clarify check
```

The check command reports missing content directories, duplicate routes, invalid OpenAPI specs, broken local Markdown links, and i18n fallback pages.

Common options:

| Option | Description | Default |
|--------|-------------|---------|
| `--root <dir>` | Project root directory | Current working directory |
| `--content <dir>` | Content directory relative to root | `source` |
| `--strict` | Treat warnings as failures | `false` |
| `--format <format>` | `text` or `json` | `text` |

---

## `clarify init`

Create the initial project files for a Clarify documentation site.

```bash
pnpm exec clarify init
```

This command scaffolds a new project and updates `package.json` with `dev` and `build` scripts.
It also adds `@clarify-labs/cli` as a dev dependency so your project is ready to run.

If you want Clarify to install dependencies automatically after scaffolding, use:

```bash
pnpm exec clarify init --install
```

If you skip `--install`, the command prints the next steps, for example:

```bash
cd <project-root>
pnpm install
clarify dev
```

Common options:

| Option | Description | Default |
|--------|-------------|---------|
| `--root <dir>` | Project root directory | Current working directory |
| `--content <dir>` | Content directory relative to root | `source` |
| `--template <name>` | `minimal`, `standard`, or `complete` template | `standard` |
| `--force` | Overwrite existing init files | `false` |
| `--install` | Install dependencies after init | `false` |
