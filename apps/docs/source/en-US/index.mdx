# Clarify

An open-source documentation publishing tool built for MDX, OpenAPI, and AI-readable knowledge bases.

Clarify keeps documentation in your repository: you maintain `source/`, OpenAPI specs, and `clarify.ts`; Clarify handles local development, route generation, React rendering, static builds, and AI-friendly content artifacts.

---

## Core highlights at a glance

| Highlight | User value |
|-----------|------------|
| **Docs as code** | Version MDX, OpenAPI, navigation, and theme configuration in Git for reviewable collaboration. |
| **MDX + OpenAPI together** | Publish product guides, tutorials, component examples, and API Reference pages in one site. |
| **Static publishing** | Generate standalone HTML for every route while keeping client-side navigation after hydration. |
| **AI-ready output** | Build `.md`, `.openapi.*`, and `llms.txt`; pages also expose copy actions for raw content and raw links. |
| **Configurable and extensible** | TypeScript config covers navigation, tabs, i18n, theme tokens, route prefixes, footer links, and plugins. |

## Quick preview

A typical Clarify docs site only needs the CLI, a config file, and content:

```bash
pnpm add -D @clarify-labs/cli
pnpm exec clarify init
pnpm dev
```

**clarify.ts**

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  title: 'My Docs',
  description: 'A documentation site powered by Clarify',
  tabs: [{ tab: 'Docs', pages: 'FileTree' }],
})
```

Then write `.md` / `.mdx` files in `source/` or add `.openapi.json` / `.openapi.yaml` specs. Clarify generates routes, pages, static HTML, raw content files, and `llms.txt`.

## Who is it for?

- **Open-source maintainers** — keep docs and code in the same repository for community contributions and version tracking.
- **Technical writers / DevRel teams** — write primarily in Markdown while using React components, API Reference pages, and structured navigation.
- **API platform teams** — publish OpenAPI specs, task-oriented guides, and AI-readable interface source files together.
- **Internal knowledge-base teams** — ship a self-hosted static docs portal that can plug into governance and search-index workflows.

- React 19
- Tailwind CSS 4 (internal styling pipeline)
- Vite 8 (internal CLI build tool)
- MDX 3
- TypeScript 5 in strict mode
