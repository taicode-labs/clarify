# Clarify

An open-source documentation publishing tool built for MDX, OpenAPI, and AI-readable knowledge bases.

Clarify keeps your documentation in the code repository: you maintain `source/`, the OpenAPI specs, and `clarify.ts`, while it handles local development, route generation, React rendering, static builds, and AI-friendly content artifacts.

---

## Core highlights at a glance

| Highlight | User value |
|------|----------|
| **Docs as code** | MDX, OpenAPI, navigation, and theme configuration all live in Git, making them naturally suited for review, versioning, and collaboration. |
| **MDX + OpenAPI together** | Product guides, tutorials, component examples, and the API reference live in the same documentation site, and the endpoint panel can display servers, authentication, parameters, examples, and request code in multiple languages. |
| **Built-in navigation and full-text search** | File routing, tabs, grouped sidebars, H2/H3 sections, and Pagefind full-text search work together so users can quickly find pages, sections, and content excerpts. |
| **Pure static publishing** | Every route generates standalone HTML while preserving the client-side navigation experience, deployable to any static hosting platform. |
| **AI-ready output** | Builds `.md`, `.openapi.*`, and `llms.txt`, and pages also offer actions to copy the raw content, open the raw link, and access the `llms.txt` link. |
| **Configurable and extensible** | TypeScript configuration covers navigation, tabs, internationalization, theme tokens, the deployment subpath, the footer, and plugins. |

## Quick preview

We recommend installing Clarify as a project dev dependency, so the CLI version is locked and reviewed alongside the repository:

```bash
npx @clarify-labs/cli init my-docs
cd my-docs
npm run dev
```

**clarify.ts**

```ts
import { defineConfig } from '@clarify-labs/cli'

export default defineConfig({
  title: 'My Docs',
  description: 'A documentation site powered by Clarify',
  tabs: [{ tab: 'Docs', pages: 'FileTree' }],
})
```

Then write `.md` / `.mdx` files in the `source/` directory, or place `.openapi.json` / `.openapi.yaml` specs there. Clarify automatically generates routes, pages, static HTML, raw content files, and `llms.txt`. For a quick trial, you can also use a global install — see the [CLI Commands Reference](/reference/cli-commands).

## Who it is for

- **Open-source maintainers** — who need docs and code in the same repository for easier community contributions and version tracking.
- **Technical writers / DevRel** — who work primarily in Markdown but also need React components, an API reference, and a unified information architecture.
- **API platform teams** — who want to publish OpenAPI specs, task-oriented guides, and AI-readable endpoint sources together.
- **Internal knowledge base teams** — who need a self-hostable static documentation portal that can integrate content governance and search indexing.
